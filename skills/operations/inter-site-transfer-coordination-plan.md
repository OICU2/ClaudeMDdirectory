# Inter-Site Transfer Coordination Plan

This skill generates a complete transfer coordination workflow with checkpoints, responsibilities, and tracking steps for moving inventory or assets between physical locations.

## Workflow Design Steps

1. **Identify Transfer Scope**
   - List all origin and destination sites involved
   - Specify asset or inventory types being transferred (SKUs, equipment, documents)
   - Determine transfer volume, weight, and any handling requirements
   - Note any regulatory, compliance, or chain-of-custody requirements

2. **Define Roles and Ownership**
   - Assign a Transfer Coordinator at each site (sender and receiver)
   - Identify who authorizes the transfer request
   - Designate a logistics owner for in-transit accountability
   - Name an escalation contact if a checkpoint fails

3. **Build the Pre-Transfer Checklist**
   - Confirm inventory counts or asset condition at origin before packing
   - Generate a Transfer Order document with unique transfer ID
   - Photograph or document item condition pre-shipment
   - Verify destination site has capacity and readiness to receive

4. **Establish Tracking Checkpoints**
   - Checkpoint 1 — Origin Dispatch: items packed, manifest signed, carrier assigned
   - Checkpoint 2 — In-Transit Scan: confirmation at midpoint or carrier pickup
   - Checkpoint 3 — Destination Arrival: receiving staff confirms package receipt
   - Checkpoint 4 — Receiving Inspection: count verified, condition checked against manifest
   - Checkpoint 5 — System Update: inventory or asset records updated in tracking system

5. **Define Exception Handling**
   - Missing items: hold receiving, flag discrepancy, notify Transfer Coordinator within 2 hours
   - Damaged items: photograph damage, quarantine affected items, file claim before accepting
   - Delayed shipment: trigger status check at 24-hour intervals, escalate after 48 hours
   - Quantity mismatch: do not close transfer until discrepancy is resolved and documented

6. **Close Out the Transfer**
   - Both sites sign off on final reconciliation report
   - Transfer ID marked complete in tracking system
   - Discrepancy log archived with transfer record
   - Post-transfer review if exceptions occurred

## Output Format

Produce a structured Transfer Coordination Plan with the following sections:

- **Transfer Summary Table**: Transfer ID, origin site, destination site, asset/inventory description, expected transfer date, assigned coordinator per site
- **Role Assignment List**: Name and contact for each role (authorizer, sender coordinator, receiver coordinator, logistics owner, escalation contact)
- **Pre-Transfer Checklist**: Numbered action items with responsible party and completion field
- **Checkpoint Tracker**: Table with columns — Checkpoint Name, Trigger Condition, Responsible Party, Required Action, Timestamp Field
- **Exception Handling Matrix**: Table with columns — Exception Type, Immediate Action, Escalation Path, Resolution Deadline
- **Sign-Off Block**: Fields for sender confirmation, receiver confirmation, and system update confirmation

Length: 1–2 pages. Use tables and numbered lists throughout. Avoid prose paragraphs. All fields should be fillable or clearly labeled for manual completion.
