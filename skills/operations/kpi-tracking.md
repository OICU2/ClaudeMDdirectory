# KPI Tracking

This skill guides teams through selecting meaningful KPIs, structuring measurement systems, and conducting regular reviews to drive operational decisions.

## KPI Definition and Measurement Workflow

**1. Clarify the Goal**
- Ask: What outcome are we trying to improve or protect?
- Map each candidate KPI directly to a specific business objective
- Reject vanity metrics that don't inform decisions

**2. Select KPIs Using SMART Criteria**
- Specific: Measures one thing clearly
- Measurable: Has a defined data source and collection method
- Actionable: A change in the number triggers a specific response
- Relevant: Tied to a goal that matters now
- Time-bound: Reviewed on a fixed cadence (weekly, monthly, quarterly)

**3. Define Each KPI Completely**
For every KPI, specify:
- **Name**: Short, unambiguous label
- **Formula**: Exact calculation (e.g., `(closed deals / qualified leads) × 100`)
- **Owner**: Person responsible for tracking and improving it
- **Data source**: Where the number comes from
- **Baseline**: Current value before any intervention
- **Target**: Numeric goal with a deadline
- **Review cadence**: How often it is formally checked
- **Alert threshold**: Value that triggers immediate attention

**4. Organize Into a KPI Stack**
- Group KPIs into tiers: Leading indicators (predictive) vs. lagging indicators (outcomes)
- Limit to 5–10 KPIs per team or function; more creates noise
- Assign one north star metric that overrides all others if there is a conflict

**5. Build the Review Ritual**
- Weekly pulse: Check leading indicators only; flag anything outside threshold
- Monthly review: Compare actuals to targets; identify root causes for gaps
- Quarterly reset: Retire KPIs that no longer reflect current goals; add new ones as strategy evolves

**6. Diagnose Underperforming KPIs**
- Is the data source reliable and consistent?
- Is the target realistic given current resources?
- Is the owner empowered to change the inputs?
- Are external factors distorting the signal?

## Output Format

Produce a structured KPI register with the following sections:

**KPI Register Table**
| KPI Name | Formula | Owner | Data Source | Baseline | Target | Cadence | Alert Threshold |
|---|---|---|---|---|---|---|---|
(one row per KPI)

**KPI Stack Summary**
- North Star Metric: [name and current value]
- Leading Indicators: [bulleted list]
- Lagging Indicators: [bulleted list]

**Review Schedule**
- Weekly: [list of KPIs + who runs the check]
- Monthly: [list of KPIs + format of review meeting]
- Quarterly: [reset process description]

**Immediate Actions**
- Up to 5 numbered steps the team should take in the next two weeks to operationalize the KPI system

Length: concise enough to fit on two printed pages. Use tables and bullets, no prose paragraphs.
