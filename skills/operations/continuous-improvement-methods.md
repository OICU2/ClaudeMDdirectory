# Continuous Improvement Methods

This skill applies Kaizen, Lean, PDCA, and related methodologies to diagnose operational problems, eliminate waste, and design repeatable improvement cycles.

## Improvement Workflow

### 1. Clarify the Problem and Scope
- Ask for a specific process, team, or workflow to analyze
- Identify the symptom vs. the root cause (use "5 Whys" if the root cause is unclear)
- Define the improvement goal: reduce time, reduce defects, reduce cost, increase throughput, or improve quality

### 2. Select the Right Methodology
Match the situation to the framework:
- **PDCA (Plan-Do-Check-Act):** General iterative improvement; good starting point for any team
- **Kaizen:** Rapid, focused improvement events; best for targeted problem areas with a team
- **Lean / Value Stream Mapping:** Eliminating waste (muda) across a process flow; identify non-value-adding steps
- **5S (Sort, Set in Order, Shine, Standardize, Sustain):** Workplace or workflow organization and discipline
- **Six Sigma (DMAIC):** Data-driven reduction of defects and variation; use when metrics are available
- **OKRs + retrospectives:** Ongoing improvement tied to goals; good for knowledge work teams

### 3. Map the Current State
- List every step in the process from trigger to output
- Mark each step as: value-adding, necessary non-value-adding, or pure waste
- Identify bottlenecks, handoff delays, rework loops, and unclear ownership
- Note what is measured today vs. what should be measured

### 4. Identify Waste Categories (Lean TIMWOODS)
Check for: **T**ransportation, **I**nventory, **M**otion, **W**aiting, **O**ver-production, **O**ver-processing, **D**efects, **S**kills underutilized

### 5. Design the Improvement Plan
- Define 1–3 high-impact changes with clear owners and deadlines
- Use a **PDCA cycle**: state the hypothesis, describe the test, define success metrics, and plan the review date
- For Kaizen events: specify duration (1–5 days), team composition, and target outcome
- Include a sustainability mechanism: standard operating procedure update, checklist, metric dashboard, or regular retrospective

### 6. Define Metrics and Review Cadence
- Baseline metric (before state)
- Target metric (after state)
- Review frequency (weekly, sprint-end, monthly)
- Escalation path if improvement stalls

## Output Format

Produce a structured improvement plan with these sections:

**Problem Statement:** One sentence describing the specific inefficiency or failure mode.

**Root Cause (5 Whys):** Numbered list of why chain, ending at root cause.

**Methodology Selected:** Name the framework and explain in one sentence why it fits.

**Current State Map:** Bulleted list of process steps with waste or delay flags noted inline.

**Waste Identified:** Categorized list using TIMWOODS or equivalent.

**Improvement Actions:** Table with columns: Action | Owner | Deadline | Expected Impact

**Success Metrics:** Baseline → Target for 1–3 measurable outcomes.

**Review Cadence:** Frequency and format (e.g., 15-min weekly standup, monthly KPI review).

**Sustainability Plan:** One paragraph describing how gains will be locked in (SOPs, training, dashboards, or scheduled retrospectives).

Length: Concise but complete—typically 400–700 words for the full plan. Use tables and bullets; avoid prose-heavy explanations unless the user asks for training content.
