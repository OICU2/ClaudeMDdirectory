# Change Management Planner

This skill creates structured change management plans covering stakeholder analysis, impact assessment, communication strategy, and phased rollout timelines when organizations need to execute transitions with minimal disruption.

## Change Management Workflow

### 1. Define the Change
- State the current state vs. desired future state clearly
- Identify the driver: why this change is necessary now
- Scope the change: teams, systems, processes, or roles affected
- Name the change sponsor (decision-maker with authority)

### 2. Stakeholder Analysis
- List all affected groups (leadership, frontline, adjacent teams, customers)
- For each group assess:
  - **Impact level**: High / Medium / Low
  - **Resistance risk**: High / Medium / Low
  - **Influence on success**: High / Medium / Low
- Identify champions (early adopters) and blockers (likely resistors)

### 3. Impact Assessment
- Map what specifically changes for each stakeholder group (workflows, tools, reporting lines, responsibilities)
- Identify dependencies: what must happen before this change can land
- Flag risks: compliance, morale, productivity dip, skill gaps
- Define success metrics: how will you know the change has stuck?

### 4. Communication Plan
- Craft core message: the "what, why, and what's in it for you" for each audience
- Define communication channels (all-hands, manager cascade, email, Slack, 1:1s)
- Set communication cadence: pre-launch, launch day, 30-day follow-up
- Prepare FAQs addressing the top 5 anticipated objections

### 5. Phased Rollout Plan
- **Phase 0 – Prepare** (2–4 weeks before): Train champions, finalize materials, confirm infrastructure
- **Phase 1 – Pilot** (select group): Validate approach, capture feedback, iterate
- **Phase 2 – Broad Rollout**: Full deployment with support structures active
- **Phase 3 – Reinforce** (30–90 days post): Monitor adoption, address laggards, celebrate wins

### 6. Resistance Management
- Anticipate top 3 resistance points with specific counterarguments
- Assign owners to monitor and respond to resistance signals
- Create a feedback loop: survey, open office hours, or anonymous input channel

### 7. Sustainment
- Define what "done" looks like (adoption rate, behavior change metrics)
- Build change into existing processes so it doesn't revert
- Schedule a 90-day retrospective to assess full adoption

## Output Format

Produce a structured Change Management Plan with these sections as headers:

1. **Change Summary** — 3-sentence overview of what, why, and scope
2. **Stakeholder Map** — Table with columns: Group | Impact | Resistance Risk | Influence | Action Needed
3. **Impact Assessment** — Bullet list of changes per group plus top 3 risks
4. **Communication Plan** — Table with columns: Audience | Message | Channel | Timing | Owner
5. **Phased Rollout Timeline** — Timeline table with Phase | Activities | Duration | Owner | Success Criteria
6. **Resistance Strategy** — 3 anticipated objections with responses and owners
7. **Success Metrics** — 3–5 measurable indicators of successful adoption

Length: Comprehensive but scannable. Use tables where data is comparative. Use bullets for lists of actions. Avoid paragraphs longer than 3 sentences. Flag any assumptions made due to missing information and list clarifying questions at the end if critical details are absent.
