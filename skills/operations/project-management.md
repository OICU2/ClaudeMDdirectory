# Project Management

Guides users through structured project planning and execution — from defining scope and milestones to tracking progress and closing out deliverables on time and within budget.

## Project Management Workflow

### 1. Clarify the Project
Before producing any plan, ask for or confirm:
- **Goal**: What does success look like?
- **Scope**: What is included — and explicitly excluded?
- **Constraints**: Deadline, budget, team size, tools available
- **Stakeholders**: Who owns it, who approves it, who is affected?

### 2. Define Phases and Milestones
Break the project into 3–6 phases appropriate to the work:
- Assign a clear deliverable to each phase
- Set a target completion date for each milestone
- Identify dependencies between phases

### 3. Build the Work Breakdown Structure (WBS)
For each phase:
- List discrete tasks (2–8 hours each where possible)
- Assign ownership (role or person)
- Flag blockers or risks per task

### 4. Identify Risks and Mitigations
List the top 3–5 risks using this format:
- **Risk**: [description]
- **Likelihood**: High / Medium / Low
- **Impact**: High / Medium / Low
- **Mitigation**: [specific action to reduce or resolve]

### 5. Define the Communication Plan
Specify:
- Reporting cadence (daily standup, weekly status, etc.)
- Status report format and owner
- Escalation path for blockers

### 6. Establish Success Criteria
Define 3–5 measurable criteria that confirm the project is complete:
- Scope delivered
- Budget variance within acceptable range
- Stakeholder sign-off received
- Quality benchmarks met

### 7. Project Closure
When closing a project, produce:
- Deliverables checklist (completed vs. planned)
- Budget actuals vs. estimate
- Lessons learned (what worked, what didn't, what to change)
- Handoff or archive instructions

## Output Format

Produce a structured project plan with the following sections, using headers and tables where appropriate:

**Project Overview**
| Field | Value |
|---|---|
| Project Name | |
| Owner | |
| Start Date | |
| Target End Date | |
| Budget | |

**Phases and Milestones**
| Phase | Deliverable | Owner | Due Date | Status |
|---|---|---|---|---|

**Work Breakdown Structure**
Bulleted list under each phase with tasks, owners, and estimated effort.

**Risk Register**
Table with columns: Risk, Likelihood, Impact, Mitigation.

**Success Criteria**
Numbered list of measurable outcomes.

**Next Immediate Actions**
3–5 specific actions the team should take in the next 48 hours, each with an owner.

Length: Comprehensive but not padded. A real, usable plan — not a template with placeholder text.
