# Process Optimization

This skill analyzes existing processes to surface waste, delays, and cost drivers, then delivers a prioritized improvement plan with measurable outcomes.

## Process Analysis Workflow

1. **Map the current state** — Ask for or reconstruct the full process step-by-step: who does what, in what order, using which tools or systems, and how long each step takes.

2. **Identify waste categories** — Evaluate each step against the 8 types of operational waste:
   - Overproduction (doing more than needed)
   - Waiting (idle time between steps)
   - Transport (unnecessary movement of information or materials)
   - Over-processing (more effort than the task requires)
   - Inventory (excess work-in-progress or backlog)
   - Motion (unnecessary actions by people)
   - Defects (errors requiring rework)
   - Unused talent (skills or people underutilized)

3. **Locate bottlenecks** — Find the step with the lowest throughput or highest queue time. That is the constraint. Improvements elsewhere will not increase overall speed until the bottleneck is addressed first.

4. **Quantify impact** — Estimate time lost, cost incurred, or error rate for each identified problem. Use the user's numbers if provided; otherwise flag where measurement is needed.

5. **Generate improvement options** — For each problem, propose at least two solutions ranging from low-effort/low-cost to higher-effort/higher-impact. Tag each with:
   - Effort level: Low / Medium / High
   - Expected impact: Incremental / Significant / Transformational
   - Implementation risk: Low / Medium / High

6. **Prioritize recommendations** — Rank improvements using impact-vs-effort. Lead with quick wins (high impact, low effort), then strategic initiatives.

7. **Define success metrics** — For each recommendation, specify what to measure and what a good outcome looks like (e.g., "reduce approval cycle from 5 days to 1 day," "cut error rate below 2%").

## Output Format

Produce a structured report with the following sections:

**Process Summary** (2–4 sentences describing the current process as understood)

**Waste & Bottleneck Analysis** (bulleted list; each item names the problem, its category, and its estimated impact)

**Improvement Recommendations** (numbered list ranked by priority; each item includes: problem addressed, proposed solution, effort level, expected impact, risk, and success metric)

**Quick Wins** (callout box or bold section highlighting the top 1–3 changes that can be implemented immediately with minimal disruption)

**Open Questions** (list of any missing information needed to refine the analysis)

Length: As long as the process complexity requires. Simple processes: 300–500 words. Complex multi-team workflows: up to 1,000 words. Use tables for side-by-side option comparison when there are four or more alternatives.
