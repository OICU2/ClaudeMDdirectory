# Performance Monitoring

This skill helps managers and teams measure progress against defined goals, surface performance gaps, and produce structured reports with actionable recommendations.

## Performance Monitoring Workflow

1. **Define the scope** — Clarify whether this is individual, team, or department-level monitoring. Identify the time period (daily, weekly, monthly, quarterly).

2. **Identify the metrics** — Extract or confirm the KPIs being tracked. Common categories:
   - Output metrics (tasks completed, units produced, tickets closed)
   - Quality metrics (error rate, customer satisfaction, rework rate)
   - Timeliness metrics (on-time delivery, response time, cycle time)
   - Engagement metrics (attendance, participation, collaboration scores)

3. **Gather baseline and targets** — Ask for or confirm:
   - The defined goal or standard for each metric
   - Current actuals or raw data
   - Historical benchmarks if available

4. **Calculate performance gaps** — For each metric:
   - Compute actual vs. target (absolute and percentage variance)
   - Flag metrics that are on track (≥95% of target), at risk (80–94%), or off track (<80%)
   - Identify trends (improving, declining, stagnant) if historical data exists

5. **Identify root causes** — For any off-track metric, prompt for or infer contributing factors:
   - Resource constraints, process bottlenecks, skill gaps, or external blockers

6. **Generate recommendations** — Produce specific, prioritized actions:
   - Quick wins (can be addressed within the current period)
   - Structural fixes (require process or resource changes)
   - Escalation items (require leadership decision)

7. **Format the output** — Produce a structured report appropriate for the audience (executive summary, manager review, or team huddle).

## Output Format

Produce a **Performance Monitoring Report** with the following structure:

---

**Performance Summary — [Team/Individual] | [Time Period]**

**Overall Status:** [On Track / At Risk / Off Track] — [1-sentence summary]

**Metric Scorecard:**

| Metric | Target | Actual | Variance | Status |
|--------|--------|--------|----------|--------|
| [Metric 1] | [Target] | [Actual] | [+/- %] | 🟢 / 🟡 / 🔴 |
| [Metric 2] | ... | ... | ... | ... |

**Key Findings:**
- [Finding 1 — specific, data-backed]
- [Finding 2 — specific, data-backed]
- [Finding 3 — specific, data-backed]

**Root Causes (Off-Track Items):**
- [Metric]: [Identified or inferred cause]

**Recommendations:**
1. [Immediate action] — Owner: [Role], Due: [Timeframe]
2. [Structural fix] — Owner: [Role], Due: [Timeframe]
3. [Escalation item if applicable]

**Next Review:** [Suggested date or cadence]

---

Keep the scorecard concise (max 10 metrics). Limit recommendations to the top 3–5 highest-impact actions. Flag any data gaps explicitly rather than making unsupported assumptions.
