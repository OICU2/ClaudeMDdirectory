# Research Participant Communication Sequence

This skill generates a full suite of staged participant communication templates—recruitment, reminder, consent confirmation, and debrief—tailored to the study type and audience so researchers can deploy professional, IRB-aware messaging immediately.

## Workflow

1. **Gather study parameters** before writing anything. Ask the researcher for:
   - Study name and brief description (what participants will do)
   - Target participant population (e.g., college students, patients, general public)
   - Compensation, if any
   - Study duration and format (in-person, online, interview)
   - PI name, institution, and contact email
   - Any sensitive topics or special consent considerations

2. **Identify which message stages are needed.** Default to all four unless the researcher specifies otherwise:
   - Stage 1: Initial recruitment
   - Stage 2: Reminder (sent 48–72 hours before session)
   - Stage 3: Consent confirmation (sent immediately after consent is obtained)
   - Stage 4: Debrief (sent after study completion)

3. **Draft each message in sequence**, applying these rules per stage:
   - **Recruitment**: Lead with the study's value to participants (not the researcher's need). State eligibility criteria clearly. Include a single, unambiguous call to action (link, reply, or contact). Keep to under 200 words.
   - **Reminder**: Reference the specific scheduled date/time/location. Restate what participants should bring or prepare. Include cancellation/rescheduling instructions. Keep to under 150 words.
   - **Consent confirmation**: Confirm participation, restate the right to withdraw at any time without penalty, provide the PI's contact for questions, and include a reference number or session ID if applicable. Keep to under 150 words.
   - **Debrief**: Thank the participant, explain the study's actual purpose (including any deception if used), provide resources if sensitive topics were covered, and share when/how results will be available. Keep to 200–300 words.

4. **Flag compliance gaps.** After drafting, note any of the following if missing from provided details:
   - IRB protocol number (should appear in recruitment and consent messages)
   - Withdrawal rights language
   - Data confidentiality statement
   - Mental health or crisis resources (required if study covers trauma, mental health, or sensitive topics)

5. **Offer customization hooks.** End each template with bracketed placeholders (`[STUDY NAME]`, `[DATE/TIME]`, `[PI EMAIL]`, etc.) so the researcher can fill in specifics without rewriting prose.

After all four templates, include a **Compliance Checklist** section listing any required elements that were flagged as missing or assumed. Each item should be one line with a checkbox format: `- [ ] Item description`.
