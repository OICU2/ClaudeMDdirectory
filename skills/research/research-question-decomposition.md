# Research Question Decomposition

This skill takes a broad or vague research question and systematically breaks it into focused, answerable sub-questions, each paired with concrete methods and data sources, so the researcher has a clear, actionable investigation plan.

## Decomposition Workflow

1. **Restate the core question** — Identify and write out the central research question in one precise sentence. If the original question is ambiguous, flag the ambiguity and propose the most reasonable interpretation before proceeding.

2. **Identify question dimensions** — Analyze the question across these axes:
   - **Definitional**: What terms need to be defined or bounded?
   - **Descriptive**: What is the current state or historical context?
   - **Causal/Explanatory**: What mechanisms or relationships need to be understood?
   - **Comparative**: Are there cases, populations, or time periods to compare?
   - **Evaluative/Normative**: What outcomes or criteria matter?
   - **Practical/Applied**: What interventions or actions are implied?

   Not every question has all dimensions — include only those that apply.

3. **Generate sub-questions** — Produce 4–8 discrete sub-questions derived from the relevant dimensions. Each sub-question must be:
   - Answerable in isolation (does not depend on answering another sub-question first)
   - Specific enough to guide a concrete search or analysis
   - Collectively exhaustive enough to address the parent question

4. **Assign methods** — For each sub-question, specify 1–2 appropriate research methods (e.g., systematic literature review, structured interviews, secondary data analysis, statistical modeling, case study, survey, ethnography, document analysis).

5. **Identify data sources** — For each sub-question, list 2–4 concrete data sources or resource types (e.g., specific databases like PubMed or JSTOR, government datasets, organizational reports, expert interviews, archival records, grey literature).

6. **Flag dependencies and sequencing** — Note any sub-questions that must be answered before others, and suggest a logical order of investigation.

7. **Surface assumptions and scope limits** — List 2–4 assumptions embedded in the original question and any explicit scope boundaries the researcher should set (geography, time period, population, discipline).

## Output Format

Produce a structured research decomposition in the following format:

---

**Core Research Question**
[One precise sentence restating the question.]

**Scope and Assumptions**
- Assumption: [statement]
- Assumption: [statement]
- Scope boundary: [statement]

**Sub-Questions and Investigation Plan**

| # | Sub-Question | Dimension | Method(s) | Data Sources |
|---|-------------|-----------|-----------|--------------|
| 1 | [Sub-question text] | [e.g., Descriptive] | [Method 1, Method 2] | [Source 1, Source 2, Source 3] |
| 2 | ... | ... | ... | ... |

**Suggested Investigation Sequence**
1. [Sub-question number and rationale for going first]
2. [Next step and why]
...

**Dependencies**
- Sub-question [X] should be completed before sub-question [Y] because [reason].

---

Length: The table should contain 4–8 rows. The sequence and dependencies sections should be brief (1–2 sentences per item). Total output should fit on one to two pages.
