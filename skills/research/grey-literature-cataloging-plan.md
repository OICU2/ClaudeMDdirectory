# Grey Literature Cataloging Plan

This skill generates a structured, actionable plan for systematically identifying, retrieving, and organizing non-peer-reviewed sources relevant to a specific research topic or question.

## Cataloging Plan Workflow

### 1. Define Scope and Source Types
- Clarify the research topic, time range, and geographic focus
- Identify relevant grey literature categories:
  - Government documents (legislation, agency reports, statistical releases)
  - International organization publications (WHO, UN, World Bank, OECD)
  - Think tank and policy institute reports
  - Industry whitepapers and trade association publications
  - Conference proceedings and technical reports
  - Dissertations and theses
  - Preprints and working papers
  - NGO and foundation reports

### 2. Map Source Repositories by Category
For each relevant category, specify concrete databases and repositories:
- **Government**: USA.gov, GovInfo, EUR-Lex, national parliament/agency websites
- **International orgs**: UN iLibrary, WHO IRIS, World Bank Open Knowledge Repository
- **Think tanks**: Brookings, RAND, Chatham House, Council on Foreign Relations
- **Industry**: specific trade association websites, corporate research portals
- **Academic grey**: ProQuest Dissertations, SSRN, arXiv, OpenDOAR
- **Aggregators**: OpenGrey, SIGLE, GreyNet International, BASE

### 3. Define Search Strategy
- List 5–10 core search terms and synonyms specific to the topic
- Specify Boolean operators and phrase combinations
- Identify date filters, language restrictions, and geographic limiters
- Note site-specific search syntax where needed (e.g., `site:gov`, `filetype:pdf`)

### 4. Establish Retrieval and Eligibility Criteria
- Define inclusion criteria: relevance threshold, recency cutoff, source credibility markers
- Define exclusion criteria: promotional materials, sources without identifiable author or institution
- Set minimum credibility standards: institutional affiliation, methodology disclosure, date of publication

### 5. Build Cataloging Structure
Create a standardized record format for each retrieved source:
- Source title, author(s), issuing organization
- Document type and category
- Publication date and version
- URL or DOI with access date
- Geographic and topical scope
- Key findings or relevance note (2–3 sentences)
- Credibility rating (institutional authority, methodology transparency)

### 6. Organize Deduplication and Quality Checks
- Cross-reference across repositories to flag duplicates
- Verify URLs and document availability
- Flag sources requiring institutional access or Freedom of Information requests
- Note sources that should be validated against peer-reviewed literature

### 7. Set Maintenance and Update Schedule
- Define a review cadence (monthly, quarterly) for new releases
- Identify alert services or RSS feeds to monitor (e.g., agency mailing lists, Google Alerts)
- Document version control for updated reports

## Output Format

Produce a structured cataloging plan with the following sections:

1. **Research Scope Summary** — 3–5 sentences defining topic, time range, geography, and purpose
2. **Source Type Inventory** — bulleted list of grey literature categories relevant to this specific topic
3. **Repository List** — table with columns: Source Type | Repository Name | URL | Search Method
4. **Search Strategy** — keyword table with terms, Boolean combinations, and filters
5. **Eligibility Criteria** — two short lists (Include / Exclude) with 4–6 criteria each
6. **Catalog Record Template** — a filled example record using a hypothetical document from the topic area
7. **Retrieval Workflow** — numbered step-by-step process (8–12 steps) tailored to the topic
8. **Update and Maintenance Plan** — brief schedule with tools and frequency
