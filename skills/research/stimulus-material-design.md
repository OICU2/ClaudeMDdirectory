# Stimulus Material Design

This skill drafts research-ready stimulus materials — concept descriptions, mock-up briefs, and scenario vignettes — calibrated for use in focus groups, usability tests, and consumer research sessions.

## Stimulus Material Workflow

1. **Clarify the research context** before drafting:
   - What is the research method? (focus group, usability test, survey, diary study)
   - Who are the target participants? (demographics, expertise level, familiarity with category)
   - What concept, product, or experience is being tested?
   - What response or behavior is the material meant to elicit?

2. **Select the appropriate stimulus format** based on context:
   - **Concept description**: For early-stage idea testing; written in plain, jargon-free prose; presents the core value proposition neutrally without overselling
   - **Mock-up brief**: For visual/design reviews; describes layout, key UI elements, user flows, and intended interactions as a written companion to a wireframe or prototype
   - **Scenario vignette**: For usability or behavioral research; places a fictional but realistic user character in a specific situation and task context

3. **Apply stimulus writing standards**:
   - Write at a 6th–8th grade reading level unless the audience requires otherwise
   - Use present tense and active voice
   - Keep language neutral — avoid superlatives, marketing language, or leading phrasing that primes participants toward a positive response
   - Ground scenarios in realistic, specific details (time of day, device, prior context) to aid imagination without over-constraining
   - For concept descriptions, state the problem, the solution, and one key differentiator — no more
   - For vignettes, name the character, establish their goal, and introduce a realistic friction point or decision moment

4. **Flag researcher considerations** at the end of each draft:
   - Note any words or phrases that may bias participant responses
   - Suggest 1–2 alternative framings if the concept is sensitive or polarizing
   - Identify what should be probed verbally after participants review the material

## Output Format

Produce one or more clearly labeled stimulus documents. Each document includes:

**Header block:**
- Material type (Concept Description / Mock-up Brief / Scenario Vignette)
- Intended method and session phase (e.g., "Focus group — early exposure round")
- Target participant profile (1 sentence)

**Body:**
- Concept Descriptions: 100–200 words, 2–3 short paragraphs, no bullet points
- Mock-up Briefs: 150–300 words with labeled sections (Purpose, Screen/Page Description, Key Elements, Intended User Action)
- Scenario Vignettes: 150–250 words, narrative paragraph format, ends on an open decision or task prompt

**Researcher Notes section:**
- 3–5 bullet points covering potential bias flags, alternative framings, and suggested follow-up probes

If the user requests multiple variants (e.g., two concept framings for A/B testing), produce each as a separate labeled block using identical structure so they are directly comparable.
