# Qualitative Research Design and Analysis

This skill guides Claude through designing rigorous qualitative research studies, developing data collection instruments, and analyzing findings to surface meaningful insights about human behavior and motivation.

## Research Design Workflow

### 1. Define the Research Question
- Clarify the central phenomenon or behavior under investigation
- Confirm the question is exploratory or explanatory (not hypothesis-testing)
- Identify what "understanding" looks like as a success condition (e.g., mental models, decision triggers, emotional drivers)

### 2. Select the Appropriate Method
- **In-depth interviews**: Best for personal experiences, sensitive topics, or individual decision-making
- **Focus groups**: Best for social norms, group dynamics, or concept reactions
- **Ethnographic observation**: Best for contextual behavior, workarounds, and unspoken practices
- **Diary/journal studies**: Best for longitudinal behavior or low-frequency events
- **Card sorting / participatory methods**: Best for mental model mapping

### 3. Define Participant Criteria
- Write inclusion and exclusion criteria tied directly to the research question
- Specify recruitment screener questions
- Determine sample size using theoretical saturation logic (typically 5–15 per segment for interviews; 2–4 groups for focus groups)
- Identify 2–3 distinct participant segments if variance is expected

### 4. Develop the Discussion Guide or Observation Protocol
- Open with warm-up questions that establish context (2–3 questions)
- Sequence from general to specific (funnel structure)
- Use open-ended probes: "Tell me more about…", "What led you to…", "Walk me through…"
- Include a critical incident technique section: ask participants to describe a specific recent instance
- Avoid leading questions, binary questions, or jargon
- Close with a "anything else" prompt and a summary question

### 5. Conduct Data Collection
- Recommend recording + transcription (with consent) over note-taking alone
- Advise on active listening techniques: mirroring, silence tolerance, laddering ("why is that important to you?")
- Flag when probing for behavior vs. attitude vs. emotion requires different techniques

### 6. Analyze the Data
- **Thematic analysis** (default): Open code → axial code → identify themes
  - Step 1: Read all transcripts; note initial impressions
  - Step 2: Generate open codes line-by-line
  - Step 3: Cluster codes into categories
  - Step 4: Identify 3–7 overarching themes with supporting evidence
- **Narrative analysis**: Use for life stories or sequential decision journeys
- **Grounded theory**: Use when building a new conceptual model from scratch
- Apply member-checking or peer debriefing to validate interpretation
- Track disconfirming evidence explicitly

### 7. Synthesize and Report
- Connect themes back to the original research question
- Distinguish between descriptive findings (what) and interpretive insights (why/so what)
- Quantify qualitative patterns cautiously: use "most," "several," "a minority"—not percentages
- Generate actionable implications for each major theme

## Output Format

Produce a structured research artifact containing:

1. **Research Brief** (3–5 sentences): Question, method rationale, participant profile, timeline estimate
2. **Discussion Guide or Protocol** (full text): Warm-up → core questions → probes → closing, formatted as a numbered script with probe indents
3. **Recruitment Screener** (if requested): 5–8 questions with pass/fail criteria marked
4. **Analysis Framework**: Codebook starter with 8–15 seed codes relevant to the topic, or a thematic map if analysis has already begun
5. **Findings Summary** (if data is provided): Themes as bold headers, 2–3 supporting quotes per theme, one "so what" implication per theme
