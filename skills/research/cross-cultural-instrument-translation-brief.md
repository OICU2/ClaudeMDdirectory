# Cross-Cultural Instrument Translation Brief

This skill generates a detailed project brief for translating and culturally adapting a research instrument, covering forward translation, back-translation, expert review, and local validation requirements.

## Translation Brief Workflow

1. **Capture instrument details**: Identify the source instrument (name, version, item count, constructs measured, original language, target language/culture, and intended population).

2. **Define translation team requirements**: Specify the number of independent forward translators needed (minimum 2), required bilingual expertise and domain knowledge, and the back-translator profile (native speaker of source language, naive to original instrument).

3. **Outline the forward translation phase**: Document each translator's independent translation, then synthesize into a single reconciled version through a committee review noting all discrepancies and resolution rationale.

4. **Specify back-translation requirements**: Back-translate the reconciled version into the source language, compare to the original for semantic equivalence, flag conceptual mismatches, and iterate until equivalence is achieved.

5. **Define expert committee review criteria**: Panel must evaluate semantic equivalence (same meaning), idiomatic equivalence (natural phrasing), experiential equivalence (cultural relevance of scenarios), and conceptual equivalence (construct validity across cultures).

6. **Specify cultural adaptation rules**: Identify items requiring cultural substitution (e.g., culturally specific examples, date formats, response scale labels), document each change with justification, and flag items recommended for deletion or replacement.

7. **Plan cognitive debriefing**: Define a pilot sample (minimum 5–10 respondents from target population), list probing questions to assess item comprehension and cultural fit, and set criteria for triggering item revision.

8. **Specify psychometric validation steps**: Confirm sample size requirements for structural validation (minimum 200–300), list analyses required (CFA for factor structure replication, measurement invariance tests, reliability coefficients, convergent/discriminant validity checks against local criterion measures).

9. **Set documentation and sign-off requirements**: Specify that all translation decisions, committee minutes, cognitive debriefing transcripts, and validation results must be archived; identify who approves each phase before proceeding.

## Output Format

Produce a structured brief with the following clearly labeled sections:

**Translation Brief: [Instrument Name] — [Source Language] to [Target Language/Culture]**

- **Instrument Overview**: Name, version, item count, constructs, source language, target language, target population
- **Translation Team**: Roles, required qualifications, conflict-of-interest restrictions
- **Phase 1 — Forward Translation**: Steps, deliverables, reconciliation process
- **Phase 2 — Back-Translation**: Steps, equivalence criteria, iteration protocol
- **Phase 3 — Expert Committee Review**: Panel composition, equivalence domains to assess, decision thresholds
- **Phase 4 — Cultural Adaptation Log**: Table template (Item | Issue | Proposed Change | Rationale | Decision)
- **Phase 5 — Cognitive Debriefing**: Sample criteria, debriefing script outline, revision triggers
- **Phase 6 — Psychometric Validation**: Sample size, analyses, equivalence tests, acceptance thresholds
- **Timeline & Milestones**: Estimated duration per phase
- **Documentation Requirements**: What must be archived and by whom

Length: 600–900 words. Use numbered phases, bullet points within each phase, and a table for the cultural adaptation log. Tone should be formal and suitable for inclusion in a research protocol or ethics submission.
