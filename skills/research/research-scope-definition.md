# Research Scope Definition

This skill produces a complete, structured research scope statement that gives a study clear direction, boundaries, and success criteria before work begins.

## Scope Definition Workflow

1. **Extract the research topic**: Identify the core subject, domain, and central question from the user's input. Ask for clarification if the topic is ambiguous before proceeding.

2. **Define research objectives**: Write 2–4 specific, measurable objectives using action verbs (analyze, compare, identify, evaluate). Each objective must connect directly to the central question.

3. **Establish boundaries**:
   - **Geographic scope**: Specify regions, countries, or "global" if applicable.
   - **Temporal scope**: Define the time period covered (e.g., 2015–2024).
   - **Thematic scope**: Name the specific dimensions being studied and explicitly exclude adjacent topics that could cause scope creep.

4. **Set inclusion criteria**: List 3–6 specific conditions a source, case, or data point must meet to be included (e.g., peer-reviewed only, minimum sample size, specific population).

5. **Set exclusion criteria**: List 3–5 explicit conditions that disqualify a source or data point, with brief rationale for each exclusion.

6. **Define deliverables**: Specify what the research will produce — report, dataset, literature review, recommendations — with format and approximate length.

7. **Identify constraints**: Note known limitations such as access restrictions, budget, time, or data availability that affect scope decisions.

8. **State out-of-scope items**: Write a short, explicit list of related topics this research will NOT address, to prevent future scope creep.

## Output Format

Produce a formatted scope statement with these labeled sections:

```
## Research Scope Statement

**Study Title:** [Concise title]

**Central Research Question:** [One clear question]

**Objectives:**
1. [Objective 1]
2. [Objective 2]
3. [Objective 3]

**Scope Boundaries:**
- Geographic: [...]
- Temporal: [...]
- Thematic: [...]

**Inclusion Criteria:**
- [Criterion 1]
- [Criterion 2]
- [Criterion 3]

**Exclusion Criteria:**
- [Criterion 1] — Rationale: [...]
- [Criterion 2] — Rationale: [...]

**Deliverables:**
- [Deliverable 1: format, length]
- [Deliverable 2: format, length]

**Known Constraints:**
- [Constraint 1]
- [Constraint 2]

**Out of Scope:**
- [Topic 1]
- [Topic 2]
- [Topic 3]
```

Length: The full scope statement should be 300–500 words. Use plain language. Avoid hedging phrases like "might include" — every item must be definitive.
