# Data Saturation Assessment

This skill analyzes qualitative data collection progress and produces a structured saturation assessment to help researchers make an evidence-based decision about whether to continue or conclude data gathering.

## Assessment Workflow

1. **Gather collection parameters** — Ask for: total number of interviews/observations completed, participant demographics or sampling strategy (purposive, theoretical, snowball), research questions, and how data is being analyzed (grounded theory, thematic analysis, IPA, etc.).

2. **Request theme emergence data** — Ask the researcher to report: how many new themes or codes emerged in the last 2–3 data collection sessions, whether existing themes are gaining depth vs. breadth, and any segments of the target population not yet sampled.

3. **Apply saturation indicators** — Evaluate against these criteria:
   - **Redundancy**: Last 2–3 sessions produced zero or near-zero new codes
   - **Replication**: Core themes confirmed across multiple participant types
   - **Coverage**: All meaningful subgroups in the target population represented
   - **Depth**: Themes have sufficient variation and negative cases documented
   - **Theoretical sufficiency** (if grounded theory): Categories are fully developed with properties and dimensions

4. **Identify saturation type** — Distinguish between:
   - *Thematic saturation*: No new themes emerging
   - *Theoretical saturation*: Categories fully developed (grounded theory)
   - *Data saturation*: Raw data repetition without analytical interpretation

5. **Flag risks** — Note if saturation may be premature due to: homogeneous sampling, insufficient negative case analysis, researcher bias toward confirming early themes, or low total N for the study scope.

6. **Form a recommendation** — State clearly: Stop collection, Continue with targeted sampling, or Conditional stop pending specific follow-up.

## Output Format

Produce a structured assessment with these sections:

**Saturation Status**: [Reached / Not Yet Reached / Approaching — 1 sentence verdict]

**Evidence Summary**:
- Theme emergence trend: [describe trajectory across recent sessions]
- Population coverage: [gaps or completeness]
- Depth indicators: [variation, negative cases, contradictions found]

**Saturation Type Achieved** (if applicable): [thematic / theoretical / data]

**Risks or Gaps**: [bulleted list of 1–4 specific concerns, or "None identified"]

**Recommendation**: [1–3 sentences — stop, continue, or conditional stop with explicit rationale]

**Suggested Next Step** (if not stopping): [specific action — e.g., "Conduct 2–3 interviews with participants from X subgroup to test negative case Y"]

Length: 200–350 words total. Use plain language suitable for a methods section or IRB response.
