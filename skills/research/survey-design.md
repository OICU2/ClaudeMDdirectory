# Survey Design

Guides the creation of well-structured surveys with clear questions, appropriate response formats, and logical flow to maximize response quality and minimize bias.

## Survey Design Process

### 1. Define Objectives
- Identify the primary research question the survey must answer
- List 3–5 specific decisions the data will inform
- Define the target respondent (role, context, prior knowledge level)
- Set a response target (e.g., 50 responses minimum for statistical validity)

### 2. Choose Survey Type
- **Quantitative**: Likert scales, multiple choice, numeric ratings — use when you need measurable, comparable data
- **Qualitative**: Open-ended questions — use when you need explanations, stories, or unexpected insights
- **Mixed**: Combine both — use when you need numbers plus context

### 3. Write Questions
Follow these rules for every question:
- One idea per question — never combine two questions with "and"
- Use neutral language — avoid leading words (e.g., "great," "problem," "obviously")
- Use specific time frames — "in the last 30 days" not "recently"
- Define ambiguous terms — if you write "often," define it
- Avoid double negatives
- Start with easy, non-sensitive questions to build momentum
- Place demographic questions last unless they are screening criteria

### 4. Select Response Formats
| Goal | Format |
|------|--------|
| Measure intensity | 5- or 7-point Likert scale |
| Rank preferences | Forced ranking (limit to 5 items) |
| Capture exact values | Open numeric field |
| Allow multiple selections | Checkbox list |
| Capture nuance | Open text (use sparingly — max 2 per survey) |
| Screen respondents | Yes/No with skip logic |

### 5. Structure the Survey
- **Opening**: 1-sentence purpose statement + estimated completion time
- **Section 1**: Screening or context questions (2–3 questions)
- **Section 2**: Core questions aligned to primary objective (5–10 questions)
- **Section 3**: Secondary or exploratory questions (2–4 questions)
- **Closing**: 1 open-ended "anything else?" + thank-you message
- Total length: 10–15 questions for most surveys; never exceed 20

### 6. Reduce Bias
- Randomize answer option order for preference questions
- Use balanced scales (equal positive and negative options)
- Avoid acquiescence bias — include questions where "disagree" is the correct answer for a satisfied user
- Pilot test with 3–5 people outside your team before launch

### 7. Add Logic and Routing
- Use skip logic to bypass irrelevant sections based on prior answers
- Use display logic to show questions only when contextually relevant
- Never route respondents into dead ends — every path must reach the closing

## Output Format

Produce a complete, ready-to-deploy survey with the following structure:

**Survey Title** — short, descriptive, respondent-facing

**Purpose Statement** — 1–2 sentences explaining why respondents are being asked and how data will be used

**Estimated Time** — realistic completion time in minutes

**Questions** — numbered list, each formatted as:
- Question number and text
- Response format label (e.g., *Likert 1–5*, *Multiple choice*, *Open text*)
- Answer options if applicable
- Skip logic instructions if applicable

**Analyst Notes** — a brief section (bullet list) explaining:
- Which questions map to which objectives
- How to interpret or analyze key questions
- Any known limitations or risks in the design

Length: Typically 1–3 pages depending on survey complexity. Prioritize clarity over completeness — a short, high-quality survey outperforms a long, exhaustive one.
