# Intercompany Pricing Policy Documentation

This skill produces structured transfer pricing policy documents for entity pairs by capturing transaction types, pricing methods, benchmark data, and regulatory compliance rationale.

## Documentation Workflow

1. **Identify Entity Pairs**: Capture the full legal names, jurisdictions, and tax residency of both the tested party and the associated enterprise for each intercompany relationship.

2. **Classify Transaction Types**: Categorize each transaction distinctly — tangible goods, services, IP licensing/royalties, financial transactions (loans, guarantees), cost allocations, or hybrid arrangements.

3. **Select Pricing Methodology**: For each transaction type, document the chosen OECD/arm's length method:
   - Comparable Uncontrolled Price (CUP)
   - Resale Price Method (RPM)
   - Cost Plus Method (CPM)
   - Transactional Net Margin Method (TNMM)
   - Profit Split Method (PSM)
   - Justify why the selected method is the "best method" over alternatives.

4. **Document Benchmarking Analysis**: Record the comparables search process — databases used (e.g., Bureau van Dijk Orbis, RoyaltyStat, PitchBook), search criteria, rejection rationale for excluded comparables, final comparable set, and the resulting arm's length range (interquartile range with median).

5. **State Tested Price or Margin**: Specify the actual intercompany price, royalty rate, interest rate, or margin applied and confirm it falls within the arm's length range.

6. **Capture Compliance Rationale**: Note applicable regulations (OECD BEPS Actions 8–10, 13; local country-specific rules; CbCR thresholds), documentation deadlines, and which entity bears documentation obligations.

7. **Record Functional Analysis**: For each entity, document functions performed, assets employed, and risks assumed (FAR analysis) that support the pricing method selection.

8. **Define Policy Terms and Conditions**: State contract terms — payment terms, currency, adjustment mechanisms (true-up provisions), and intercompany agreement reference numbers.

9. **Flag Risk Areas**: Identify transactions with thin margins, hard-to-value intangibles, financial transactions in low-rate jurisdictions, or entities in countries with aggressive transfer pricing enforcement.

## Output Format

Produce a structured policy document with the following sections for each entity pair:

```
INTERCOMPANY TRANSFER PRICING POLICY
Entity Pair: [Entity A] ↔ [Entity B]
Effective Date: [Date] | Review Date: [Date]

1. ENTITY OVERVIEW
   - Entity A: [Legal name, jurisdiction, tax ID, role]
   - Entity B: [Legal name, jurisdiction, tax ID, role]

2. TRANSACTION SUMMARY TABLE
   | Transaction Type | Annual Volume (est.) | Currency | Direction |
   |-----------------|---------------------|----------|-----------|

3. PRICING METHODOLOGY
   - Selected Method: [Method name]
   - Best Method Justification: [2-4 sentences]
   - Rejected Methods: [List with reasons]

4. BENCHMARKING SUMMARY
   - Database(s): [Names]
   - Search Date: [Date]
   - Comparables Found/Accepted: [X of Y]
   - Arm's Length Range: [Lower quartile] – [Upper quartile] (Median: [X])

5. APPLIED PRICE/MARGIN
   - Intercompany Rate/Price: [Value]
   - Position in Range: [Percentile or confirmation]

6. FUNCTIONAL ANALYSIS SUMMARY
   | Party   | Key Functions | Key Assets | Key Risks |

7. COMPLIANCE NOTES
   - Applicable Rules: [OECD/local regulations]
