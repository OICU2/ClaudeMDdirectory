# Solopreneur Case Study Pipeline Builder

This skill evaluates completed client projects for case study potential, ranks them by persuasiveness, and produces a concrete interview guide and writing plan to systematically convert them into published case studies.

## Case Study Pipeline Workflow

### Step 1: Project Inventory Intake
Ask the user to list completed client projects (last 12–24 months) with brief details for each:
- Client industry and company size
- The core problem or goal they hired you to solve
- What you delivered
- Any measurable results (revenue, time saved, growth %, etc.)
- Whether the client relationship ended positively
- Likelihood the client would participate in an interview (high/medium/low)

If the user has fewer than 5 projects, prompt for any freelance, consulting, or pro bono work that produced a tangible outcome.

### Step 2: Score Each Project on Five Criteria
Rate each project 1–3 on each dimension (max score: 15):

1. **Result Clarity** — Are outcomes specific and measurable? (vague = 1, directional = 2, hard numbers = 3)
2. **Audience Relevance** — Does this client mirror the user's ideal future client? (no = 1, partial = 2, exact = 3)
3. **Story Drama** — Was there a real before/after transformation or obstacle overcome? (flat = 1, moderate = 2, compelling = 3)
4. **Client Accessibility** — Can the client be quoted, interviewed, or named? (no = 1, partial/anonymous = 2, full participation = 3)
5. **Competitive Differentiation** — Does the project showcase a unique method, speed, or expertise? (generic = 1, somewhat = 2, strongly = 3)

Produce a ranked table showing all projects, their scores per criterion, and total score.

### Step 3: Recommend a Tiered Pipeline
Organize scored projects into three tiers:

- **Tier 1 (Score 12–15): Build first** — Full named case studies with client quotes
- **Tier 2 (Score 8–11): Build second** — Anonymized case studies or mini case snapshots
- **Tier 3 (Score below 8): Skip or hold** — Flag for future use if results improve or client opens up

Recommend producing 2–3 Tier 1 case studies before moving to Tier 2.

### Step 4: Generate a Client Interview Guide
For each Tier 1 project, produce a tailored 10-question interview guide covering:
- Situation before hiring (context, pain, urgency)
- Why they chose you over alternatives
- What the working process felt like
- Specific results achieved and how they measured them
- Unexpected benefits or surprises
- What they would tell a peer considering hiring you

Include a short email template the user can send to request the interview, framed as a "quick success story feature."

### Step 5: Produce a Writing Plan
Create a per-case-study writing plan with:
- Recommended structure: Headline → Client Snapshot → The Problem → The Approach → The Results → The Takeaway → CTA
- Word count target: 500–800 words for full case studies, 150–250 for mini snapshots
- Suggested headline formula: "[Outcome achieved] for [Client Type] in [Timeframe/Context]"
- Distribution channels to publish on (website, LinkedIn, proposal decks, email sequences)
- A sequenced 30/60/90-day publishing schedule based on tier priority

### Step 6: Identify Missing Results
If a project has strong story potential but lacks hard metrics, prompt the user to:
- Reach out to the client for a brief update on outcomes
- Estimate conservatively using available data (hours saved, % improvement
