# Solo Operator Annual Leave Revenue Protection Plan

This skill builds a concrete pre-leave revenue and operations plan for solopreneurs, covering income pre-building, retainer structuring, automation setup, and client communication — activated when someone needs to take time off without their income collapsing.

## Revenue Protection Workflow

### Step 1 — Establish the Leave Parameters
Ask (or infer from context):
- How many days/weeks off?
- What is current average monthly revenue?
- What income is already recurring vs. project-based?
- What is the planned leave start date? (Calculate runway in weeks.)

### Step 2 — Revenue Audit & Gap Analysis
- Calculate expected revenue loss during leave (project-based work that won't happen)
- Identify recurring revenue already locked in (retainers, subscriptions, passive income)
- Define the **protection gap**: the dollar amount that needs to be pre-built or replaced

### Step 3 — Pre-Build Revenue Strategies
Recommend specific tactics based on the operator's business type:

**Pre-sell or front-load work:**
- Offer clients a discounted pre-pay block for work to be delivered after return
- Complete and invoice deliverables early, banking payment before leave
- Run a limited-time offer (e.g., "book now for post-leave delivery") to fill the calendar on return

**Retainer structuring:**
- Convert any active project clients to a monthly retainer before leave
- Offer a 3-month retainer with a 5–10% discount to lock in revenue
- Define a "light retainer" option: async check-ins only, no new deliverables during leave

**Productised or passive revenue:**
- Identify any existing IP (templates, guides, courses) that can be sold while offline
- Suggest one digital product to create before leave if none exists
- Set up a simple evergreen offer with automated checkout and delivery

### Step 4 — Automation Stack for Time Away
- **Email auto-responder**: Draft a professional out-of-office with expected return date, emergency escalation contact (if any), and a self-serve resource link
- **Scheduling buffer**: Block calendars to prevent bookings during leave; open slots only for the week after return
- **Payment automation**: Ensure all outstanding invoices are sent before departure; set up automatic payment reminders
- **Delivery automation**: If any async work will drip out during leave, schedule it via tool (e.g., scheduled email, Notion, Loom delivery)
- **Social/content**: Schedule any content needed to maintain visibility; recommend 2–3 posts per week minimum

### Step 5 — Client Communication Plan
Draft a tiered communication sequence:

**Tier 1 — Active retainer clients (2–3 weeks before leave):**
- Notify of leave dates
- Confirm what is/isn't covered during absence
- Set expectations for response times

**Tier 2 — Active project clients (1–2 weeks before leave):**
- Deliver or checkpoint all in-progress work before departure
- Agree on a pause or handoff plan

**Tier 3 — Leads and prospects:**
- Set auto-reply or landing page noting next available date
- Offer to hold a slot for post-return start

### Step 6 — Return Ramp Plan
- Schedule a "re-entry week" with lighter load
- List the top 3 priority actions for Day 1 back (e.g., invoice follow-ups, check-in emails, pipeline review)
- Set a post-leave revenue target for the first 30 days to rebound quickly
