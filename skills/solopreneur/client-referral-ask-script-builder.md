# Client Referral Ask Script Builder

This skill creates a warm, specific, non-salesy referral request script a solopreneur can send or say to a satisfied client at the ideal moment in the relationship.

## Referral Script Building Process

**Step 1: Gather context before writing anything**

Ask the user for:
- Client's name and what service/project was delivered
- A concrete result or win the client experienced (specific numbers or outcomes preferred)
- The relationship warmth level: formal/professional, friendly, or close
- Preferred delivery method: email, text/DM, or verbal/phone
- Whether they want to ask for a specific person, a general introduction, or a testimonial alongside the referral

If the user skips details, make reasonable assumptions and state them clearly at the top of the output.

**Step 2: Identify the right framing**

Choose the script angle based on relationship warmth:
- Formal/professional → lead with the client's result, frame referral as helping the user serve more people like them
- Friendly → lead with gratitude, make it conversational and light
- Close → skip formality, be direct and personal, treat it as a favor between people who trust each other

**Step 3: Write the script using these rules**

- Open with a genuine, specific callback to the work done together (not generic praise)
- State the result the client achieved — use their language if provided
- Make the ask in one sentence, clearly but softly — no hedging chains ("if you happen to maybe know someone who might possibly...")
- Give the client an easy out ("no pressure at all, just wanted to put it on your radar")
- Close with a forward-looking line that reinforces the relationship, not just the transaction
- Keep total length under 150 words for email/text, under 90 words for verbal

**Step 4: Add a follow-up note**

After the script, include a one-line note on timing: when in the client lifecycle to send this (e.g., within 2 weeks of project completion, after receiving positive feedback, after a milestone result is confirmed).

## Output Format

Produce the following sections in order:

**Assumptions (if any):** Bullet list of anything filled in without user input.

**Referral Script:**
A ready-to-send script in the exact format requested (email, text, or verbal). Use the client's name. No placeholders like [INSERT NAME] — write it as if it's real and ready to go.

**Timing Note:**
One sentence on when to send or say this for maximum receptivity.

**Optional Variation:**
A one-sentence tweak that adjusts tone (slightly warmer or slightly more direct) if the user wants to A/B test their approach.
