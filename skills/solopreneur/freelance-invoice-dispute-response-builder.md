# Freelance Invoice Dispute Response Builder

This skill drafts professional, evidence-backed written responses to client payment disputes or invoice objections, drawing on contract terms, scope agreements, and delivery records the user provides.

## Dispute Response Workflow

### Step 1: Gather Context
Ask the user for the following if not already provided:
- The client's specific objection or dispute claim (exact wording if possible)
- Relevant contract or agreement terms (scope of work, payment terms, revision limits, acceptance criteria)
- Proof of delivery (files sent, emails confirming receipt, project milestones completed)
- Invoice amount, due date, and current payment status
- Prior communication history (any approvals, sign-offs, or change requests)

### Step 2: Classify the Dispute Type
Identify which category the dispute falls into:
- **Scope creep claim** — client says work delivered doesn't match what was agreed
- **Quality objection** — client claims deliverables are substandard
- **After-the-fact discount request** — client accepts work but contests the price post-delivery
- **Non-delivery claim** — client denies receiving work
- **Contract misunderstanding** — client misinterprets payment terms or deadlines

### Step 3: Build the Response
Structure the response using these principles:
- Open with acknowledgment of the dispute without conceding fault
- Cite specific contract clauses, agreed scope, or documented approvals directly
- Present delivery evidence factually and chronologically
- State the amount owed and the original due date clearly
- Set a firm but professional deadline for resolution
- Offer one reasonable path forward only if appropriate (e.g., a call to review evidence), never unprompted discounts
- Close with consequences of non-payment (late fees per contract, collections, legal escalation) stated matter-of-factly, not as threats

### Step 4: Tone Calibration
Apply the appropriate tone based on dispute stage:
- **First response**: Professional, firm, factual — assume good faith
- **Second response / escalation**: Formal, direct, no conversational softening
- **Final notice**: Terse, legal in register, explicit about next steps

### Step 5: Review for Weak Language
Remove or replace:
- Apologetic openers ("I'm sorry you feel this way")
- Hedging phrases ("I think," "maybe," "I believe")
- Unprompted concessions ("I can lower the amount if needed")
- Emotional language in either direction

## Output Format

Produce a ready-to-send email or letter with:

1. **Subject line** — clear and professional (e.g., "Re: Invoice #[NUMBER] — Response to Payment Dispute")
2. **Salutation** — formal by default
3. **Body** — 3–5 short paragraphs:
   - Paragraph 1: Acknowledge the dispute, state the purpose of the response
   - Paragraph 2: Reference contract terms and agreed scope with specifics
   - Paragraph 3: Present delivery evidence (dates, file names, confirmation emails)
   - Paragraph 4: Restate the amount owed and payment deadline
   - Paragraph 5 (if needed): Outline next steps or consequences
4. **Closing** — professional sign-off
5. **Optional footnote** — list of attachments the user should include (contract excerpt, delivery confirmation, prior approval emails)

Length: 200–400 words for first response; 100–200 words for escalation or final notice.
Format: Plain prose, no bullet points in the email body itself.
Tone: Confident, factual, professional — never hostile, never apologetic.
