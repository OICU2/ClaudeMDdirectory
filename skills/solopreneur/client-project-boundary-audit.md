# Client Project Boundary Audit

Systematically reviews one or more active client engagements to identify where scope, time, and communication boundaries are being crossed or are at risk, then delivers a prioritized action plan.

## Audit Workflow

### 1. Gather Project Data
Ask the user to provide for each client under review:
- Original agreed scope (what was promised in writing or verbally)
- Current actual scope (what is being done now)
- Contracted hours or budget vs. hours/money spent to date
- Communication channel and frequency expectations vs. reality
- Start date, deadline, and current status

If the user doesn't have exact numbers, prompt them to estimate and flag those estimates clearly.

### 2. Run the Three-Boundary Check

**Scope Boundary:**
- Compare original deliverables to current deliverables line by line
- Flag any task, deliverable, or responsibility not in the original agreement
- Classify each addition as: Formal Change Order, Informal Agreement, or Undiscussed Assumption

**Time Boundary:**
- Calculate hours or effort spent vs. contracted
- Identify if any single week or phase consumed disproportionate time
- Flag recurring time sinks (endless revision cycles, excessive meetings, "quick questions")

**Communication Boundary:**
- Identify channels being used vs. agreed channels
- Note response time expectations the client appears to hold vs. what was agreed
- Flag after-hours contact, multi-platform pings, or demands for immediate responses

### 3. Score Each Violation
Rate each identified violation:
- **Red**: Active boundary breach causing measurable harm (revenue loss, burnout, deadline risk)
- **Yellow**: Pattern forming that will become Red within 2–4 weeks if unaddressed
- **Green**: Minor drift, monitor only

### 4. Generate Corrective Actions
For each Red and Yellow item, produce one specific action:
- A script or talking point for a client conversation
- A contract clause or email template to reset expectations
- A process change to prevent recurrence (e.g., stop answering Slack after 6pm, require written briefs before revisions)

## Output Format

Produce a structured audit report in this format:

---
**CLIENT BOUNDARY AUDIT: [Client Name or Project Name]**
**Audit Date:** [date]

**Overall Health:** 🔴 Critical / 🟡 At Risk / 🟢 Healthy

---

**SCOPE VIOLATIONS**
| Issue | Classification | Severity | Action |
|-------|---------------|----------|--------|
| [specific issue] | [Formal/Informal/Undiscussed] | 🔴/🟡/🟢 | [one-line action] |

**TIME VIOLATIONS**
| Issue | Hours Lost | Severity | Action |
|-------|-----------|----------|--------|

**COMMUNICATION VIOLATIONS**
| Issue | Frequency | Severity | Action |
|-------|-----------|----------|--------|

---

**TOP 3 PRIORITY ACTIONS**
1. [Most urgent action — include exact script or template if a conversation is needed]
2. [Second action]
3. [Third action]

**WATCH LIST** (Green items to monitor)
- [item]

---

If auditing multiple clients, produce one table per client, then a cross-client summary ranking which client relationship needs attention first.
