# Solo Operator Downtime Revenue Plan

This skill builds a concrete, personalized plan for sustaining baseline revenue during planned or unplanned downtime using retainers, digital products, deferred billing, or hybrid structures.

## Discovery Phase

Before generating a plan, Claude must establish:

1. **Downtime type** — Is this planned (vacation, parental leave, sabbatical) or unplanned contingency planning?
2. **Downtime duration** — Days, weeks, or months?
3. **Current revenue model** — Hourly, project-based, retainer, product sales, or mixed?
4. **Monthly baseline need** — The minimum revenue required to cover essential expenses
5. **Existing assets** — Any current retainer clients, email list, digital products, courses, templates, or recurring subscriptions?
6. **Client relationship style** — Do clients expect active access or are they outcome-focused?

If the user hasn't provided these, ask for them in a single grouped message before proceeding.

## Plan Construction Rules

Build the plan across three layers:

### Layer 1 — Protect Existing Revenue
- Identify which current client relationships can be converted to retainers or maintenance agreements
- Draft a retainer framing: what the client gets (availability, priority access, monthly deliverable) vs. what you stop doing (ad hoc billing)
- Recommend deferred billing structures only if cash flow permits: invoice now for work delivered after return
- Flag any active project deadlines that must be renegotiated or completed before downtime begins

### Layer 2 — Create Passive or Low-Touch Revenue
- Identify 1–3 digital product ideas directly derived from the operator's existing expertise (templates, guides, SOPs, mini-courses, audits)
- Prioritize products that can be built in under 10 hours and sold without active fulfillment
- Recommend a single platform for delivery (Gumroad, Notion, Lemon Squeezy, etc.) based on simplicity
- Suggest a simple pre-downtime launch sequence: existing audience email, one social post, pinned link

### Layer 3 — Set a Floor and a Ceiling
- Calculate the revenue floor: the minimum monthly amount covered by retainers + passive products
- Identify the gap between the floor and the baseline need
- Suggest one contingency action if the gap is significant (e.g., a "done-for-you" package sold in advance, a waitlist deposit model, or a one-time bulk project before leave)
- Set a ceiling: the point at which no additional work should be accepted before or during downtime

## Communication Templates

Provide ready-to-send language for:
- A retainer conversion pitch (2–3 sentences, client-facing)
- An out-of-office or availability notice for existing clients
- A one-line digital product launch message for email or social

## Output Format

Produce a structured plan with the following sections:

1. **Situation Summary** — 2–3 sentences restating the operator's downtime scenario and baseline need
2. **Revenue Protection Actions** — Numbered list of 3–5 specific actions to take before downtime, each with a one-line rationale
3. **Recommended Revenue Structure** — A simple table showing: Source | Type | Estimated Monthly Value | Setup Effort
4. **Projected Revenue Floor** — Single line showing the expected minimum monthly income if the plan is executed
5. **Gap & Contingency** — One paragraph addressing any shortfall and the recommended fix
6. **Ready-to-Use Copy** — The three communication templates, clearly labeled

Total output length: 400–700 words. Use plain language. No jargon. Prioritize what can be done in the next 7 days.
