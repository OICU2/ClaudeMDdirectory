# Freelance Subcontractor Briefing Template

Produces a complete, professional briefing document a solo operator can hand directly to a subcontractor to align on project scope, standards, deadlines, and communication norms before work begins.

## Briefing Generation Workflow

1. **Gather project essentials** — Ask the user for: project name, client context (industry, goals), the subcontractor's specific role, key deliverables, timeline/deadlines, and any non-negotiables. If the user provides partial info, fill reasonable gaps and flag assumptions clearly.

2. **Define deliverables with precision** — List each deliverable with: a clear name, description, format/spec, due date, and acceptance criteria. Vague deliverables ("good design work") must be made concrete ("3 homepage mockups in Figma, 1920px wide, using provided brand kit").

3. **Document project standards** — Capture: file naming conventions, tool stack (where files live, how tasks are tracked), brand or style guidelines, revision policy (how many rounds, turnaround expectation), and quality benchmarks.

4. **Set communication norms** — Specify: preferred channel (Slack, email, etc.), expected response time, check-in cadence (daily standup, weekly async update, etc.), who the subcontractor reports to, and escalation path for blockers.

5. **Include legal and operational basics** — Add sections for: confidentiality expectations, IP ownership (work-for-hire or otherwise), invoice/payment process, and rate/hours agreed upon.

6. **Close with a confirmation step** — End the document with a simple acknowledgment section the subcontractor can sign or reply to confirm they've read and understood the brief.

## Output Format

Produce a single, ready-to-send briefing document with the following sections in order:

```
# Subcontractor Briefing: [Project Name]
**Role:** [Subcontractor's function]
**Prepared by:** [Solo operator's name or "Your Name"]
**Date:** [Today's date or placeholder]

---

## 1. Project Overview
[3–5 sentences: client context, project goal, why this subcontractor is involved]

## 2. Your Role & Scope
[Bullet list of responsibilities. What is in scope. What is explicitly out of scope.]

## 3. Deliverables
| Deliverable | Description | Format/Spec | Due Date | Acceptance Criteria |
|-------------|-------------|-------------|----------|---------------------|
| ...         | ...         | ...         | ...      | ...                 |

## 4. Project Standards
- File naming: [convention]
- Tools & file locations: [links or descriptions]
- Brand/style guidelines: [link or summary]
- Revision policy: [X rounds within Y days]

## 5. Communication Norms
- Primary channel: [tool]
- Response time expectation: [X hours]
- Check-in cadence: [frequency + format]
- Point of contact: [name/role]
- Blocker escalation: [process]

## 6. Legal & Payment
- Confidentiality: [expectation]
- IP ownership: [work-for-hire / other]
- Rate: [amount and structure]
- Invoicing: [how and when to invoice]
- Payment timeline: [net-X days]

## 7. Acknowledgment
By replying "Confirmed" or signing below, [Subcontractor Name] confirms they have read and agree to the terms of this brief.

_Signature / Date: ___________________________
```

Length: 1–2 pages when rendered. Use plain language. Flag any section where the user needs to fill in missing details using `[PLACEHOLDER: description]`.
