# Solopreneur Canned Response Library Builder

This skill builds a structured, ready-to-use library of customizable email and message templates covering the most common client inquiries, objections, and admin requests a solopreneur faces, organized for fast retrieval and easy personalization.

## Workflow

### Step 1 — Profile the Solopreneur
Ask (or infer from context) three things:
- **Business type**: What service or product do they sell? (e.g., freelance design, coaching, consulting, digital products)
- **Communication channels**: Where do they receive messages? (email, Instagram DMs, LinkedIn, contact forms)
- **Tone**: Professional and formal? Warm and casual? Direct and brief?

If the user doesn't provide details, default to a general service-based solopreneur with a warm-professional tone.

### Step 2 — Identify the Core Response Categories
Build templates across these seven categories (always include all seven):

1. **New Inquiry / "How much do you charge?"** — First response to a cold or warm lead asking about services and pricing
2. **Discovery Call Invitation** — Following up on interest to book a call
3. **Proposal Follow-Up** — Nudging a prospect who has gone quiet after receiving a proposal
4. **Pricing Objection** — Responding to "that's more than I expected" or "can you do it cheaper?"
5. **Scope Creep Boundary** — Professionally addressing requests that fall outside the agreed project
6. **Project Delay / Timeline Update** — Notifying a client of a delay or requesting more time
7. **Testimonial / Referral Request** — Asking a happy client for a review or referral after project completion

### Step 3 — Write Each Template
For every template:
- Open with a **subject line** (for email) or **message opener** (for DMs/chat)
- Write the full template body using `[BRACKETS]` for every variable the user must fill in (e.g., `[Client Name]`, `[Project Type]`, `[Your Rate]`, `[Deadline]`)
- Keep length appropriate to context: inquiry replies = 3–5 sentences; boundary-setting = 4–7 sentences; requests = 2–4 sentences
- End each template with a single, clear call to action
- Write in the tone specified in Step 1

### Step 4 — Add a Personalization Checklist
After each template, include a 3-bullet "Before You Send" checklist:
- Which variables to fill in
- One optional line to add for extra personalization
- One thing to remove if the situation doesn't apply

### Step 5 — Create a Quick-Reference Index
At the top of the library output, produce a numbered index listing all seven template names with a one-line description of when to use each.

## Output Format

Produce a single, complete document structured as follows:

```
# [Business Type] Canned Response Library
*Tone: [Tone] | Channels: [Channels]*

---

## Quick-Reference Index
1. [Template Name] — [One-line use case]
2. ...

---

## Template 1: [Category Name]
**When to use:** [One sentence]
**Subject / Opener:** [Subject line or opening line]

---
[Full template body with [BRACKETED] variables]
---

**Before You Send:**
- Fill in: [list variables]
- Add for warmth: [optional personal line]
- Remove if: [condition]

---
[Repeat for all 7 templates]
```

Total length: 600–1,000 words for the full library. Each template body: 50–120 words. Use plain text formatting
