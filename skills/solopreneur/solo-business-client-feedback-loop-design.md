# Solo Business Client Feedback Loop Design

This skill builds a complete, repeatable feedback system tailored to a solopreneur's specific service model — covering when to ask, what to ask, how to review responses, and how to turn insights into action without creating administrative overhead.

## Feedback Loop Design Process

### Step 1: Clarify the Business Context
Before designing anything, gather:
- What type of service is delivered (project-based, retainer, coaching, done-for-you, etc.)
- Average project duration and natural end points
- Current client volume per month or quarter
- Whether the solopreneur has any existing follow-up habit (even informal)
- Primary goal: testimonials, improvement, retention, or all three

### Step 2: Define the Trigger Moment
Identify the single best moment to request feedback — not too early, not too late:
- Project-based work: 3–5 days after final delivery, once the client has had time to use/review the output
- Retainer/ongoing services: End of each billing cycle or every 60–90 days
- Coaching/consulting: Within 24 hours of a milestone session or program completion
- Name this moment explicitly so it can be calendared or automated

### Step 3: Design the Feedback Collection Method
Choose the right format based on client relationship and volume:
- **High-touch clients (< 5/month):** Short personal email with 3 open-ended questions
- **Medium volume (5–20/month):** Single-page form (Typeform, Google Form) with 5 questions max
- **High volume (20+/month):** Automated post-project email sequence with embedded 1-click rating + optional comment

Draft the exact questions using this framework:
1. One outcome question: "Did we accomplish what you were hoping for?"
2. One experience question: "Was there any point where you felt confused, unsure, or underserved?"
3. One improvement question: "What's one thing that would have made this even better?"
4. One forward-looking question: "Would you work with me again / refer me to someone?" (NPS-style)
5. Optional testimonial prompt: "If you'd be willing to share a short testimonial, I'd be grateful."

### Step 4: Build the Review Ritual
Design a recurring review habit — not real-time reaction:
- Set a weekly 20-minute block (e.g., Friday afternoon) to read all feedback received that week
- Use a simple log: spreadsheet or Notion table with columns — Date, Client, Score/Sentiment, Key Quote, Action Item
- Categorize each response: Positive / Constructive / Urgent
- Flag any feedback that reveals a repeated pattern (same issue mentioned 2+ times = systemic problem)

### Step 5: Create the Action Decision Tree
Define what happens with each category of feedback:
- **Positive:** Extract usable testimonial language, send thank-you reply, add to testimonials folder
- **Constructive:** Log the specific improvement, decide whether to address in next project or update process now
- **Urgent (client dissatisfied):** Respond within 24 hours, offer a specific remedy, note root cause
- **Patterns (repeated issues):** Schedule a dedicated fix — update onboarding doc, scope template, delivery checklist, etc.

### Step 6: Close the Loop with Clients
Design a one-sentence follow-up for each response type:
- After positive feedback: Thank them and mention you've noted their kind words
- After constructive feedback: Acknowledge the point and share one concrete change you're making
- After urgent feedback: Confirm the resolution and what you've changed going forward

This step is optional but builds extraordinary client trust.
