# Discovery-to-Proposal Conversion Tracker

This skill analyzes discovery call and proposal data to surface conversion ratios, identify drop-off patterns, and give solo operators specific fixes for their pitch and qualification process.

## Tracking & Analysis Workflow

1. **Collect raw data** — Ask the operator to provide for the last 30–90 days: total discovery calls held, number of proposals sent, number of proposals that converted to paid projects, and any notes on why calls didn't result in proposals.

2. **Calculate core metrics**
   - Discovery-to-Proposal Rate = (Proposals Sent ÷ Discovery Calls) × 100
   - Proposal-to-Close Rate = (Closed Deals ÷ Proposals Sent) × 100
   - Overall Call-to-Close Rate = (Closed Deals ÷ Discovery Calls) × 100
   - Flag if Discovery-to-Proposal Rate is below 40% (qualification problem) or Proposal-to-Close Rate is below 30% (pitch/pricing problem)

3. **Identify drop-off stage** — Determine where the biggest leak is:
   - **Pre-proposal drop-off**: Calls happen but proposals aren't sent → qualification, fit, or follow-through issue
   - **Post-proposal drop-off**: Proposals sent but not closing → pricing, value framing, or follow-up issue
   - **Both stages leaking**: Systemic process problem

4. **Diagnose root cause** — Ask targeted questions based on the drop-off stage:
   - Pre-proposal: "What % of calls end with a clear next step scheduled?" / "Are you disqualifying on budget before the call?"
   - Post-proposal: "How long after the call do you send the proposal?" / "Do you follow up more than once?"
   - Both: "Do you have a defined ideal client profile you screen against?"

5. **Generate specific fixes** — Produce 3–5 prioritized action items tied directly to the identified drop-off stage, not generic advice.

6. **Set a benchmark target** — Define a realistic 30-day improvement goal based on current numbers (e.g., raise Discovery-to-Proposal Rate from 25% to 45%).

## Output Format

Produce a structured report with these sections:

**Conversion Snapshot**
- Table showing: Discovery Calls | Proposals Sent | Deals Closed | Discovery→Proposal % | Proposal→Close % | Overall %

**Drop-Off Diagnosis**
- 2–3 sentences identifying where the biggest leak is and the likely cause

**Root Cause Findings**
- Bulleted list of 2–4 specific patterns observed (e.g., "Proposals sent 5+ days after calls correlate with low close rates")

**Action Plan**
- Numbered list of 3–5 concrete fixes with the exact change to make (e.g., "Send proposal within 24 hours of call — block 90 minutes post-call to write it")

**30-Day Target**
- Single line stating the specific metric to move and the goal number

Keep the full output under 400 words. Use plain language, no jargon. If data is incomplete, state exactly what's missing and ask for it before proceeding.
