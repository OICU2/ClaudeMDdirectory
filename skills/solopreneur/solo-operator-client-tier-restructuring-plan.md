# Solo Operator Client Tier Restructuring Plan

This skill analyzes a solopreneur's current client roster and produces a concrete tiered service restructuring plan that increases average revenue per client and eliminates or upgrades low-value engagements.

## Client Audit & Tier Design Workflow

**Step 1: Extract Current Client Data**
Ask the user to list each client with: monthly revenue, average hours spent per month, type of work delivered, and one-word description of the relationship (easy/draining/neutral). If they don't have exact hours, ask for a rough estimate. Do not proceed without at least revenue and hours for each client.

**Step 2: Calculate Effective Hourly Rate**
For each client, divide monthly revenue by monthly hours to get effective hourly rate (EHR). Flag any client where EHR is below 50% of the user's stated or implied target rate. These are Tier Restructure Candidates.

**Step 3: Score Each Client on Three Axes**
Score 1–3 on each axis:
- **Revenue potential**: Can this client realistically pay more or expand scope?
- **Energy cost**: How draining is the relationship and communication overhead?
- **Strategic value**: Does this client provide referrals, portfolio value, or market access?

Composite score = Revenue potential + Strategic value − Energy cost. Higher score = higher priority client.

**Step 4: Design Three Service Tiers**
Based on the scoring, define three tiers tailored to the user's business:
- **Tier 1 (Core/Retainer)**: Top-scoring clients. Define a premium package with expanded deliverables, priority access, and a price floor 20–40% above current rates.
- **Tier 2 (Standard)**: Mid-scoring clients. Define a clearly scoped package with defined response times, deliverable limits, and a modest rate adjustment if EHR is below target.
- **Tier 3 (Project/Exit)**: Low-scoring or below-EHR clients. Offer a reduced-scope project-only arrangement or begin a graceful offboarding sequence.

**Step 5: Build the Migration Plan**
For each client, assign a migration action:
- Upgrade: Move to Tier 1 with a rate conversation within 30 days.
- Retain: Move to Tier 2 with adjusted scope at renewal.
- Convert: Offer Tier 3 project pricing; do not renew retainer.
- Offboard: Send closing communication within 60 days.

Set a 90-day migration timeline with specific decision points at Day 30 (Tier 1 conversations done), Day 60 (Tier 2 adjustments communicated), Day 90 (Tier 3 exits complete).

**Step 6: Draft Client Communication Scripts**
Write a short, specific script for each migration action type:
- Rate increase conversation opener for Tier 1 upgrades
- Scope clarification email for Tier 2 transitions
- Project-only offer for Tier 3 converts
- Professional exit message for offboarded clients

Scripts must be direct, warm, and non-apologetic. No filler phrases like "I just wanted to reach out."

## Output Format

Produce a structured document with the following sections:

**1. Client Audit Table**
Markdown table with columns: Client Name, Monthly Revenue, Monthly Hours, EHR, Energy Score, Revenue Potential, Strategic Value, Composite Score, Assigned Tier.

**2. Tier Definitions**
Three labeled sections (Tier 1, Tier 2, Tier 3) each containing: what's included, price floor, access/response parameters, and who currently qualifies.

**3. Migration Action Plan**
Bullet list per client: name
