# Client Boundary Enforcement Script Generator

This skill produces ready-to-send, professionally worded message templates that help solo operators enforce agreed-upon scope, hours, and communication terms with clients who routinely exceed them — without damaging the relationship.

## Boundary Assessment and Script Generation Workflow

1. **Identify the violation type** — Determine which boundary is being crossed:
   - Scope creep (requests outside agreed deliverables)
   - Hour overruns (work requests beyond contracted hours)
   - Communication overreach (after-hours contact, excessive check-ins, wrong channels)
   - Payment or timeline disputes tied to expanded requests
   - Repeated violations after a previous conversation

2. **Gather context before writing** — If not already provided, ask:
   - What was the original agreement (contract terms, SOW summary, or verbal agreement)?
   - What specifically is the client asking for or doing?
   - Has this happened before, and was it addressed?
   - What outcome does the operator want (redirect, upsell, firm no, relationship preservation)?

3. **Calibrate the tone** — Match message firmness to the situation:
   - **First offense / soft overstep** → Warm redirect with gentle reframe
   - **Repeat offense / clear scope violation** → Direct, factual, boundary-setting
   - **Chronic violator / relationship at risk** → Firm, professional, consequence-aware

4. **Write the enforcement message** using this structure:
   - **Acknowledge** — Briefly recognize what the client wants or communicated
   - **Anchor** — Reference the agreed terms without accusation ("As outlined in our agreement...")
   - **Redirect or Offer** — Either decline clearly or present a paid path forward
   - **Close** — End warmly but without leaving ambiguity

5. **Include a short internal note** explaining why the message is worded as it is, so the operator understands the strategy and can adapt it.

6. **Offer a follow-up option** — After the main template, provide one alternative version at a higher or lower firmness level so the operator can choose the best fit.

## Output Format

Produce the following for each request:

**Situation Summary** (1–2 sentences confirming understanding of the boundary issue)

---

**Message Template — [Tone Level: Soft / Direct / Firm]**

> [Ready-to-send message, written in first person as the solo operator. 3–6 sentences. No placeholders left blank — use logical assumptions or clearly marked brackets like [project name] only when truly unknown.]

---

**Why This Works** (2–3 sentences explaining the communication strategy used)

---

**Alternate Version — [Higher/Lower Firmness]**

> [Second ready-to-send message at a different tone level, same structure]

---

**Optional Add-On** (only if relevant): A one-line suggested subject line for email, or an opening line for a verbal conversation version.
