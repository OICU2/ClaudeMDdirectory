# Launch-Week Revenue Projection

Builds a structured revenue model for a new offer launch by walking through list size, price point, and conversion rate assumptions to produce realistic best/base/worst case projections.

## Projection Workflow

1. **Gather inputs** — Ask for the following if not already provided:
   - Total reachable audience (email list size, social following, or both)
   - Offer price point (or price range if undecided)
   - Launch window length (default to 7 days if unspecified)
   - Launch mechanic (email sequence, webinar, open cart, waitlist drop, etc.)

2. **Apply conversion benchmarks** — Use these industry-standard ranges for warm audiences:
   - Cold/low-engagement list: 0.5%–1% conversion
   - Average/moderately engaged list: 1%–3% conversion
   - Hot/highly engaged list: 3%–5% conversion
   - Webinar or live launch funnel: add 0.5%–1% lift over baseline
   - Ask the user to self-assess their list warmth if unclear

3. **Build three scenarios** — Calculate revenue for each:
   - **Worst case**: low-end conversion rate × list size × price
   - **Base case**: mid-range conversion rate × list size × price
   - **Best case**: high-end conversion rate × list size × price

4. **Identify the break-even threshold** — Calculate the minimum number of sales needed to cover any launch costs (ads, tools, contractor help) if the user provides them; otherwise note $0 cost baseline.

5. **Flag unrealistic expectations** — If the stated revenue goal requires a conversion rate above 5% on a cold list or above 8% on any list, name it directly and explain why the math is strained.

6. **Offer lever analysis** — For each scenario, show what happens if the user changes one variable:
   - Raises price by 20%
   - Doubles list size (e.g., via a pre-launch referral push)
   - Adds a payment plan (model at 1.3× unit volume, 0.85× collected revenue in launch week)

7. **State assumptions explicitly** — List every assumption used so the user can adjust inputs and rerun.

## Output Format

Produce a clean projection table followed by narrative commentary:

**Section 1 — Inputs Summary**
Bullet list of all inputs used (list size, price, launch mechanic, list warmth, launch window).

**Section 2 — Revenue Projection Table**
Markdown table with columns: Scenario | Conversion Rate | Units Sold | Gross Revenue
Include worst, base, and best case rows.

**Section 3 — Lever Analysis**
Three short bullet points showing the impact of the price increase, list growth, and payment plan levers on base-case revenue.

**Section 4 — Recommendation**
2–4 sentences. State which scenario is most likely given the inputs, what the single highest-leverage action is before launch, and whether the stated goal is achievable without changing the model.

**Section 5 — Assumptions**
Numbered list of every assumption baked into the model.

Total output length: 300–500 words. Use plain language. No jargon. No motivational filler.
