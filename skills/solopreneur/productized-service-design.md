# Productized Service Design

Transforms a vague or custom service into a structured, sellable package with defined scope, pricing, and deliverables that can be sold repeatedly without custom proposals.

## Service Packaging Workflow

### Step 1: Extract the Core Service
Ask the user to describe what they currently do for clients. Identify:
- The single transformation or outcome they deliver
- The most common version of the work (the 80% case)
- What they always end up doing regardless of client size

### Step 2: Define Strict Scope Boundaries
Create a crisp "what's included / what's not included" split:
- List exactly 4–7 concrete deliverables (e.g., "3 SEO-optimized blog posts," not "content")
- Set a fixed turnaround time (e.g., "delivered in 10 business days")
- List 3–5 explicit exclusions to prevent scope creep
- Specify the number of revision rounds (usually 1–2)

### Step 3: Name the Package
Create a results-oriented package name using this formula:
- [Outcome] + [Service Type] or [Timeframe] + [Result]
- Examples: "Launch-Ready Brand Kit," "30-Day SEO Jumpstart," "Conversion Copy Sprint"
- Avoid generic names like "Starter Plan" or "Basic Package"

### Step 4: Build 2–3 Pricing Tiers
Structure tiers around scope expansion, not feature unlocks:
- **Tier 1 (Entry):** Narrowest scope, fastest delivery, lowest price — targets price-sensitive buyers
- **Tier 2 (Core):** The "right fit" option — anchor this as the recommended tier
- **Tier 3 (Premium):** Expanded scope, faster turnaround, or done-for-you add-ons
- Price each tier so Tier 2 feels like the obvious choice (price Tier 3 at ~2.5x Tier 1)

### Step 5: Write the Positioning Statement
Craft one sentence per tier: "For [client type] who [situation], this package delivers [outcome] in [timeframe] for $[price]."

### Step 6: Identify the Sales Page Structure
Outline the five sections the user needs to sell this package:
1. Problem statement (1 sentence)
2. Package name + outcome promise
3. Deliverables list
4. Pricing tiers table
5. FAQ (scope, revisions, timeline, who it's not for)

## Output Format

Produce a structured document with these exact sections:

**Package Name:** [Name]

**One-Line Description:** [Who it's for + what they get + timeframe]

**What's Included:**
- [Deliverable 1]
- [Deliverable 2]
- [Deliverable 3–7]

**What's Not Included:**
- [Exclusion 1–5]

**Pricing Tiers:**
| Tier | Name | Scope | Turnaround | Price |
|------|------|-------|------------|-------|
| 1 | [Name] | [scope] | [days] | $[X] |
| 2 | [Name] | [scope] | [days] | $[X] |
| 3 | [Name] | [scope] | [days] | $[X] |

**Positioning Statement per Tier:** [3 sentences, one per tier]

**Suggested FAQ Questions:** [5 questions the user should answer on their sales page]

**Next Action:** One concrete next step (e.g., "Write the sales page," "Set up a Stripe payment link," "Create a Notion client intake form")

Length: 400–600 words. Use tables and bullet lists. No paragraphs
