# Client Testimonial Leverage Plan

This skill takes raw client testimonials and builds a deployment map showing exactly which testimonial goes where, why, and how to adapt it for maximum conversion at each stage of the buyer journey.

## Testimonial Analysis Workflow

### Step 1: Collect and Categorize Input
Ask the user to paste their testimonials (or describe them). If they haven't shared any, ask for 3–10 real client quotes or feedback snippets before proceeding.

### Step 2: Extract the Core Signal from Each Testimonial
For each testimonial, identify:
- **The specific result or outcome mentioned** (revenue, time saved, emotional shift, transformation)
- **The before-state implied** (pain, doubt, hesitation the client had)
- **The objection it neutralizes** (e.g., "too expensive," "I can do it myself," "will this work for me?")
- **The buyer persona it speaks to** (beginner, skeptic, budget-conscious, experienced buyer)
- **The emotional tone** (relief, excitement, confidence, surprise)

### Step 3: Map Each Testimonial to the Buyer Journey
Assign each testimonial to one of three stages:

**Awareness Stage** — Use testimonials that describe the problem vividly or the moment the person realized they needed help. Best for: blog posts, social media, cold outreach, podcast appearances.

**Consideration Stage** — Use testimonials that compare alternatives, explain why they chose you, or address specific doubts. Best for: sales pages, email sequences, webinar slides, proposal decks.

**Decision Stage** — Use testimonials that focus on results, ROI, and ease of working with you. Best for: checkout pages, discovery call follow-ups, pricing pages, DM conversations.

### Step 4: Match Testimonials to Top Sales Objections
Identify the 4–6 most common objections for the user's offer (ask if unknown). Then assign the best-fit testimonial to each objection:

| Objection | Best Testimonial Match | Placement |
|-----------|----------------------|-----------|
| "It's too expensive" | [testimonial about ROI or value] | Pricing page, sales call |
| "I don't have time" | [testimonial about quick wins or speed] | Email sequence |
| "Will this work for me?" | [testimonial from similar client] | Sales page, DMs |
| "I've tried things before" | [testimonial about difference/uniqueness] | Cold outreach |

### Step 5: Write Deployment Instructions for Each Testimonial
For each testimonial, produce a short action block:
- Where to place it (exact page, email number, post type)
- How to introduce it (suggested lead-in sentence)
- Whether to use the full quote or pull a power snippet (bolded key phrase)
- Any edits needed for clarity or impact (with original preserved)

### Step 6: Flag Testimonial Gaps
Note which objections, personas, or journey stages have no testimonial coverage. Suggest 2–3 prompts the user can send to past clients to collect the missing social proof.

## Output Format

Produce a structured **Testimonial Leverage Plan** with these sections:

1. **Testimonial Inventory Table** — One row per testimonial with: client (first name or initialism), core result, objection neutralized, buyer persona, emotional tone, journey stage

3. **Objection Coverage Matrix** — Table showing each objection and which testimonial covers it
