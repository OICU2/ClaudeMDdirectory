# Client Communication Boundary Scripts

Generates ready-to-send email templates and conversation scripts for solopreneurs who need to set, clarify, or enforce professional boundaries around response times, revision limits, and scope changes without damaging client relationships.

## Boundary Script Workflow

1. **Identify the boundary situation** — Determine which category applies:
   - Response time / after-hours contact
   - Revision limit reached or exceeded
   - Scope creep / out-of-contract request
   - Payment terms / late payment boundary
   - Communication channel enforcement (e.g., "stop texting me, use email")

2. **Assess the relationship stage** — Ask if this is:
   - A new client (set expectations proactively)
   - An existing client who hasn't been told the boundary yet (introduce gently)
   - A repeat offender who has ignored previous requests (firm reinforcement)

3. **Determine the tone register needed**:
   - Warm & educational: first time raising the issue
   - Neutral & professional: second mention
   - Direct & non-negotiable: boundary has been violated repeatedly

4. **Draft the script using this structure**:
   - **Acknowledge** the client's underlying need or urgency (one sentence max)
   - **State the boundary** clearly and without apology
   - **Explain the reason** briefly (optional, keep to one sentence)
   - **Offer the path forward** — what they should do instead or when they can expect a response
   - **Close warmly but definitively**

5. **Add a subject line** for email versions that signals professionalism, not conflict (e.g., "A quick note on our workflow" not "Regarding your behavior").

6. **Provide a short version and a long version** when the situation warrants different levels of detail.

7. **Flag any legal or contractual language** the solopreneur should reference from their own contract (don't fabricate contract terms — prompt them to insert their own).

## Boundary-Specific Rules

- **Never draft an apology** for having the boundary — remove "sorry" and "I apologize" unless genuinely warranted
- **Avoid passive-aggressive phrasing** — scripts should be assertive, not hostile
- **Revisions script**: always state the exact number of revisions included per contract before explaining the overage policy
- **Scope creep script**: always include language that reframes the request as a new opportunity ("this sounds like a great add-on project")
- **After-hours script**: include the specific response window so clients know when to expect replies
- **Never write "as per my last email"** — rephrase to something non-combative like "to recap what we discussed"

## Output Format

Produce the following for each request:

**Subject Line (email only):** One line, professional tone

**Short Version (2–4 sentences):**
A brief, direct message suitable for replying to a Slack message, text, or quick email thread.

**Full Version (1–3 short paragraphs):**
A complete email or script with acknowledgment, boundary statement, reason (if applicable), and clear next step.

**Optional add-on:**
A one-sentence follow-up message to send if the client pushes back or does not comply after the initial message.

**Tone label:** State which tone register was used (Warm & Educational / Neutral & Professional / Direct & Non-Negotiable) so the solopreneur can adjust if needed.

Format all scripts in plain text, ready to copy-paste. Use bracketed placeholders like [CLIENT NAME], [NUMBER OF REVISIONS], [YOUR RESPONSE HOURS] for any details the solopreneur must customize.
