# Freelance Project Milestone Planner

This skill structures any client project into concrete milestones with deadlines, deliverables, and payment triggers so freelancers protect their cash flow and prevent scope creep from the start.

## Milestone Planning Workflow

**Step 1: Extract Project Parameters**
Ask for (or infer from context):
- Total project scope and deliverables
- Agreed budget and payment structure (fixed, retainer, hourly)
- Project start date and final deadline
- Client industry and communication preferences
- Any known risk areas (vague requirements, large revisions, stakeholder approval delays)

**Step 2: Define Project Phases**
Divide the project into 3–6 phases depending on complexity. Each phase must map to a tangible output the client can see or approve. Standard phase structure:

1. **Discovery / Kickoff** — requirements locked, brief signed, assets received
2. **Foundation / First Draft** — first major deliverable ready for review
3. **Revision Cycle(s)** — structured rounds with a defined limit (e.g., 2 rounds max)
4. **Final Delivery** — approved files, handoff documentation, access transfers
5. **Post-Launch / Wrap** (if applicable) — support window, training, or reporting

**Step 3: Set Deadlines with Buffer**
- Assign a client-facing deadline to each milestone
- Add 20–30% internal buffer on phases dependent on client feedback
- Flag phases with external dependencies (third-party approvals, asset delivery from client) as high-risk and add explicit due dates for client inputs

**Step 4: Define Deliverables Per Milestone**
For each milestone, list:
- Exactly what will be handed over (file types, formats, platforms, number of items)
- What constitutes "complete" (approval criteria)
- What is explicitly out of scope for that phase

**Step 5: Attach Payment Triggers**
Assign a payment percentage or fixed amount to each milestone. Standard structures:
- **3-milestone projects**: 40% upfront / 30% at midpoint / 30% on final delivery
- **4–5 milestone projects**: 30% upfront / 25% mid / 25% late / 20% final
- Never attach the final payment to "client satisfaction" — attach it to delivery of the approved final files
- Flag which milestones require payment before work continues

**Step 6: Identify Scope Guard Rails**
List 2–4 explicit scope boundaries per project:
- What triggers a change order (additional rounds, new features, platform changes)
- How change orders are priced (hourly rate or flat fee add-on)
- What happens if client misses an input deadline (project pauses, timeline shifts by equivalent days)

## Output Format

Produce a milestone plan with the following structure:

---

**Project:** [Name/Type]
**Total Budget:** [Amount]
**Start Date → Final Deadline:** [Date range]

---

**MILESTONE TABLE**

| # | Milestone Name | Deadline | Deliverables | Payment Trigger | Amount |
|---|---------------|----------|--------------|-----------------|--------|
| 1 | Kickoff & Discovery | [Date] | Signed brief, asset checklist, project roadmap | Invoice due before work begins | $X (XX%) |
| 2 | [Phase Name] | [Date] | [Specific outputs] | Invoice on delivery | $X (XX%) |
| … | … | … | … | … | … |

---

**SCOPE BOUNDARIES**
- In scope: [list]
- Out of scope: [list]
- Change order rate: $X/hr or $X flat per addition

**RISK FLAGS**
- [Any phases with client-dependency delays]
- [Revision round limits]
- [What happens if client ghosts or misses deadlines
