# Content Gap Analysis

This skill systematically compares existing content against target topics, audience needs, and competitor coverage to surface specific gaps and produce a prioritized list of content to create or expand.

## Analysis Workflow

1. **Inventory existing content** — Collect titles, URLs, topics, and content types from the user. If they can't provide a full list, ask for their main content categories or a representative sample.

2. **Define the target scope** — Clarify the subject domain, target audience, and goals (SEO traffic, user education, sales enablement, etc.). Ask if they have a keyword list, competitor URLs, or topic taxonomy to compare against.

3. **Map coverage** — For each topic or keyword in the target scope, determine:
   - Fully covered (dedicated, thorough content exists)
   - Partially covered (mentioned but not explored in depth)
   - Not covered (entirely absent)

4. **Analyze competitor coverage** — If competitor URLs or names are provided, identify topics they address that the user's content does not. Note content formats competitors use (guides, comparisons, tutorials, FAQs) that are missing from the user's library.

5. **Score each gap** — Assign a priority score based on:
   - Audience relevance (high / medium / low)
   - Estimated search demand or business value
   - Effort to create (quick fill vs. major piece)
   - Competitive pressure (competitors have strong coverage)

6. **Generate recommendations** — For each high-priority gap, specify: the topic, recommended content type, target audience intent, and a one-sentence rationale.

## Output Format

Produce a structured report with these sections:

**Coverage Summary**
- Total topics audited
- Fully covered: N
- Partially covered: N
- Not covered: N

**Gap Table**
A markdown table with columns: Topic | Gap Type (missing/thin) | Priority (High/Med/Low) | Recommended Action | Rationale

**Top 5 Priority Gaps**
Numbered list. Each entry includes:
- Topic name
- Why it matters (audience need or competitive reason)
- Recommended content type (e.g., "2,000-word how-to guide," "comparison page," "FAQ section")
- Suggested title or angle

**Quick Wins**
Bullet list of 3–5 existing pieces that can be expanded or updated with minimal effort to close partial gaps.

Keep the Gap Table scannable. Limit rationale column to one sentence per row. Total output should be skimmable in under 3 minutes.
