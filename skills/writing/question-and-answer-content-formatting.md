# Q&A Content Formatting

This skill takes raw interview transcripts, notes, or conversation logs and transforms them into polished, publication-ready Q&A format with consistent voice, trimmed filler, and editorial coherence.

## Formatting Workflow

1. **Identify speakers** — Distinguish interviewer from respondent. Label consistently as **Q:** and **A:** or use names if provided (e.g., **Interviewer:** / **[Name]:**).

2. **Edit questions** — Rewrite interviewer questions to be concise, clear, and direct. Remove filler phrases ("So, um, I was wondering if maybe you could tell us about..."), false starts, and redundant framing. Each question should stand alone without context from the one before.

3. **Edit answers** — Preserve the respondent's voice and meaning while removing verbal tics ("you know," "like," "honestly," "I mean"), run-on structures, and off-topic tangents. Break overly long answers into readable paragraphs. Do not change the substance or add new claims.

4. **Maintain consistent tense and tone** — Standardize verb tense throughout. Match the register of the publication context (casual, professional, academic) based on cues in the source material or any instructions provided.

5. **Sequence and structure** — Reorder Q&A pairs only if explicitly asked or if the original order is clearly illogical. Flag any reordering done.

6. **Handle incomplete answers** — If a response is cut off, unclear, or missing, insert a bracketed note: `[Response unclear — original transcript incomplete]`.

7. **Add formatting anchors** — If the Q&A is long (8+ exchanges), suggest a subheading structure to break it into thematic sections (e.g., **On Career Beginnings**, **On Industry Trends**).

## Output Format

Produce the formatted Q&A as clean, ready-to-publish text:

- Each exchange starts with **Q:** followed by the edited question on its own line
- **A:** follows immediately below with the edited response
- A single blank line separates each Q&A pair
- Subheadings (if applicable) appear as `## [Section Title]` above the relevant exchange
- At the end, include a brief editorial note in italics listing any significant changes made (reordering, omitted tangents, unclear sections flagged)

Example structure:

```
**Q:** What drew you to this field initially?

**A:** I stumbled into it, honestly. I was studying economics and took an elective in urban planning that completely reoriented me. By my third year I knew I wasn't going back.

**Q:** How has the industry shifted since you started?

**A:** The data side has exploded. When I began, most decisions were gut-driven. Now every proposal needs a model behind it.

---
*Editorial note: Two off-topic exchanges about the respondent's commute were omitted. One answer was split into two paragraphs for readability.*
```
