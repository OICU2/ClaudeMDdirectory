# Writing Scope of Work Builder

This skill drafts a professional, detailed scope of work document for writing projects and activates whenever someone needs to define deliverables, timelines, and terms for a writing engagement.

## Scope of Work Drafting Workflow

1. **Gather project essentials** — Before drafting, collect or infer: project type (blog posts, white paper, web copy, etc.), client/party names, total word count or volume, deadline or delivery schedule, number of revision rounds, and any style or format requirements.

2. **Define the project overview** — Write a 2–4 sentence project summary that identifies the client, the writer, the purpose of the content, and the intended audience or platform.

3. **List deliverables explicitly** — Enumerate every deliverable with specifics:
   - Content type and title (if known) or description
   - Word count range per piece (e.g., 800–1,000 words)
   - File format (Google Doc, Word, PDF, etc.)
   - Quantity (e.g., 4 blog posts, 1 white paper)

4. **Specify revision rounds** — State the exact number of included revision rounds (e.g., "Two rounds of revisions per deliverable"). Define what counts as a revision versus a new request. Note the rate or process for out-of-scope revisions.

5. **Set the timeline** — Provide a milestone-based schedule:
   - Project kick-off date
   - First draft delivery date(s)
   - Client feedback deadline(s)
   - Revised draft delivery date(s)
   - Final delivery date

6. **State what is out of scope** — List at least 3 explicit exclusions (e.g., SEO keyword research, image sourcing, publishing/uploading, additional formats beyond what's specified).

7. **Include approval and payment terms** — Add a brief section covering: how the client approves deliverables, what triggers final payment, and a note that scope changes require a written amendment.

8. **Add a signature block** — Include placeholder fields for client name, writer name, date, and signature lines.

## Output Format

Produce a structured document with clearly labeled sections using bold headers. Use bullet points for deliverables and exclusions; use a simple table or numbered list for the timeline. Total length should be 400–700 words depending on project complexity. The document should be ready to copy, paste, and send to a client with minimal editing. Fill in any unknown details with clearly marked placeholders in brackets (e.g., [Client Name], [Delivery Date]).
