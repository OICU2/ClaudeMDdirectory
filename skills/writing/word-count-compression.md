# Word-Count Compression

This skill trims an over-length draft to a specific word count target by cutting redundancy and tightening language while preserving every core argument, data point, and the author's original voice.

## Compression Workflow

1. **Confirm the target.** Identify the requested word count. If none is stated, ask before proceeding. Note the current word count and calculate the reduction percentage.

2. **Audit before cutting.** Read the full draft and mentally tag:
   - **Protect:** core arguments, named data points, statistics, quotes, proper nouns, unique voice markers
   - **Cut candidates:** throat-clearing intros, redundant restatements, filler phrases ("it is important to note that"), adverb stacks, passive constructions that inflate length

3. **Cut in priority order:**
   - Remove sentence-level redundancy first (repeated ideas stated twice)
   - Collapse verbose phrases to single words ("in order to" → "to", "due to the fact that" → "because")
   - Trim weak adverbs and adjectives that add length without meaning
   - Consolidate multi-sentence points into one tight sentence
   - As a last resort, identify the least-essential supporting example and cut it entirely — flag this explicitly to the user

4. **Preserve voice.** Match the author's register (formal/casual), sentence rhythm, and characteristic word choices. Do not normalize idiosyncratic style.

5. **Verify the count.** After rewriting, confirm the output lands within ±2% of the target word count. If it overshoots, do another tightening pass before delivering.

6. **Flag any losses.** If a point, example, or data item had to be removed (not just condensed), call it out explicitly at the end so the author can decide whether to restore it.

## Output Format

Deliver the following in order:

**Compression Summary** (3 lines max):
- Original word count → Target word count → Actual output word count
- Reduction percentage achieved
- Any content removed (not just condensed), listed as bullet points

**Compressed Draft:**
The full rewritten text, ready to use, with no inline annotations or tracked changes unless the user requests them.
