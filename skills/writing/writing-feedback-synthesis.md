# Writing Feedback Synthesis

This skill analyzes competing editorial feedback from multiple sources, identifies contradictions, resolves them using principled criteria, and produces a concrete revision plan that honors the author's voice and goals.

## Synthesis Workflow

1. **Collect all feedback** — Ask the user to paste or summarize every piece of feedback received, labeled by source (e.g., Editor A, Manager, Peer Reviewer 2).

2. **Extract discrete suggestions** — Break each reviewer's comments into individual, atomic suggestions. Label each with: source, target passage, and requested change.

3. **Group by document section or theme** — Cluster suggestions that address the same paragraph, argument, or stylistic element so conflicts become visible.

4. **Identify contradictions explicitly** — Flag any cases where two or more reviewers request opposing changes (e.g., "cut this section" vs. "expand this section"). State the conflict plainly: *Reviewer A wants X; Reviewer B wants Y.*

5. **Apply resolution criteria in order:**
   - **Author intent first** — Does one option better serve the stated purpose or audience of the piece?
   - **Scope of role** — Weight feedback according to the reviewer's domain (e.g., a copy editor's grammar note outranks a stakeholder's grammar note; a subject-matter expert's factual correction outranks a stylistic preference).
   - **Evidence over opinion** — Prefer feedback backed by a specific reason over vague preference.
   - **Consistency with the whole** — Choose the option that creates fewer ripple conflicts elsewhere in the document.
   - **Escalate if unresolvable** — Flag conflicts that require the author to make a judgment call or consult a decision-maker.

6. **Assign priority tiers to all suggestions:**
   - **P1 — Must address:** Factual errors, structural problems, audience mismatches, requests from primary decision-makers.
   - **P2 — Should address:** Clarity improvements, well-reasoned stylistic changes, consensus suggestions.
   - **P3 — Consider or decline:** Stylistic preferences with no clear rationale, suggestions that conflict with author intent.

7. **Draft the revision plan** — For each P1 and P2 item, write one actionable instruction the author can execute.

## Output Format

Produce a structured revision plan with the following sections:

**Conflicts Identified**
A numbered list of each contradiction found, naming the sources and summarizing both positions in one sentence each.

**Resolution Decisions**
For each conflict: the chosen direction, a one-sentence rationale, and whether it is a firm recommendation or an author judgment call.

**Prioritized Revision Checklist**
A flat, ordered checklist grouped by P1 / P2 / P3. Each item is one actionable sentence (e.g., "Shorten the methodology section from ~400 words to ~200 words, cutting the historical background paragraph entirely."). Items within each tier are ordered by document sequence.

**Suggestions to Decline**
A short list of feedback not incorporated, with a one-line reason for each (e.g., "Reviewer C's request to switch to passive voice — conflicts with target publication's style guide and author's established voice.").

**Escalation Items**
Any conflicts that could not be resolved without additional information or a stakeholder decision, with a suggested question the author can ask to break the tie.

Total output length should match document complexity: expect 300–600 words for a short piece with moderate feedback, up to 1,000 words for complex multi-stakeholder documents.
