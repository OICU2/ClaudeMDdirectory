# Structural Outline Reverse Engineering

This skill deconstructs any finished piece of writing into its reusable structural blueprint so the user can replicate the format with new content.

## Reverse Engineering Workflow

1. **Read for architecture, not content** — Identify the macro structure first: how many sections, what each section accomplishes functionally (hook, context, argument, counterargument, resolution, CTA, etc.), and how sections transition.

2. **Label each block by function** — Replace actual content with its functional role. "In 2008, the economy collapsed" becomes "[Inciting statistic or event that establishes stakes]". Strip all specifics; keep the rhetorical move.

3. **Identify the sentence-level patterns** — Note recurring sentence structures, paragraph length norms, how claims are supported (anecdote → data → insight, or claim → example → implication), and voice/register choices.

4. **Map the argumentative or narrative logic** — Identify the underlying progression: problem/solution, chronological, compare/contrast, zoom-in/zoom-out, listicle with through-line, etc. Name it explicitly.

5. **Extract repeating micro-patterns** — Flag any repeated structural units (e.g., every section opens with a question, every example follows a 3-sentence arc) and codify them as rules.

6. **Build the reusable template** — Assemble all findings into a fill-in-the-blank outline a writer can use to write a new piece in the same structural style from scratch.

## Output Format

Produce three clearly labeled sections:

**1. Structural Diagnosis (short prose, 100–150 words)**
Name the piece type, its overall logic/progression, intended audience effect, and any notable structural choices that define its voice or persuasiveness.

**2. Annotated Block Breakdown**
A numbered list of every structural block in order. Each entry includes:
- Block label (e.g., "Block 3: Tension Escalation")
- Functional description (what it does rhetorically)
- Length/format note (e.g., "2–3 short paragraphs" or "single punchy sentence")
- One bracketed placeholder showing what content would go here

**3. Reusable Fill-in-the-Blank Template**
A clean, copy-paste-ready outline using only bracketed placeholders. No original content. Structured so any writer can drop in new material and produce a piece with the same architecture. Include a one-line "Template Rules" note summarizing 2–3 constraints the writer must follow to preserve the structure's integrity.
