# Sales Page Writing

Produces complete, long-form sales pages using proven direct-response frameworks that move readers from skeptical to sold.

## Core Workflow

### 1. Extract the Essentials First
Before writing, identify or ask for:
- **Product/service**: What is being sold, what it does
- **Target buyer**: Who they are, their pain, their dream outcome
- **Price point**: Determines how much copy and trust-building is needed
- **Top 3 objections**: Why someone would NOT buy
- **Proof available**: Testimonials, results, credentials, guarantees

### 2. Build the Page in This Order

**HEADLINE**
- Lead with the biggest transformation or result
- Format: "[Achieve Dream Outcome] Without [Biggest Fear/Objection]"
- Write 3 variants and select the strongest

**HOOK (Opening 3–5 sentences)**
- Open with the reader's pain or frustration, not the product
- Make them feel seen before you sell

**AGITATE THE PROBLEM**
- Deepen the pain with specifics
- Name the failed alternatives they've already tried
- Quantify the cost of doing nothing

**INTRODUCE THE SOLUTION**
- Present the product as the inevitable answer
- Do NOT list features yet — sell the concept first

**CREDIBILITY / ORIGIN STORY**
- Who made this and why should the reader trust them
- Keep it relevant — connect backstory to the buyer's problem

**WHAT YOU GET (Features + Benefits)**
- For each feature: state it, then explain the benefit with "which means..."
- Use bullet points; lead each bullet with the outcome, not the feature

**SOCIAL PROOF**
- Insert testimonials near objections they address
- Use results-based quotes: "I went from X to Y in Z time"
- If no testimonials exist, note where placeholders should go

**OBJECTION BUSTING**
- Address each major objection directly with a header ("But what if I'm not tech-savvy?")
- Reframe objections as reasons to buy, not reasons to pause

**THE OFFER STACK**
- List everything included with perceived value for each item
- Make the total value obvious before revealing price

**PRICE REVEAL + JUSTIFICATION**
- Anchor price against the cost of the problem or alternatives
- Frame as investment, not expense

**GUARANTEE**
- State terms clearly and boldly
- Remove all purchase risk language ("If you're not satisfied for any reason...")

**CALL TO ACTION**
- One clear CTA button/phrase repeated 3–4 times down the page
- Use action language: "Get instant access," "Start today," "Claim your spot"

**P.S. CLOSE**
- Restate the transformation + urgency + guarantee in 2–3 sentences
- This is for skimmers — treat it as a mini sales letter

### 3. Objection Handling Rules
- Never ignore an objection — burying it amplifies it
- Address price before the reader asks by justifying value first
- Handle "I need to think about it" with urgency (deadline, scarcity, cost of delay)
- Handle "Does this work for me?" with specific use-case examples

### 4. Tone and Style Rules
- Write at an 8th-grade reading level
- Use short paragraphs (1–3 sentences max)
- Never use passive voice in CTAs
- Replace every "we" with "you" where possible
- Avoid superlatives without proof ("best," "greatest") — use specifics instead

## Output Format

Deliver a complete sales page as structured markdown with clearly labeled sections using bold headers matching the framework above. Include:

- **Length**: 1,500–3,500 words depending on price point (higher price = more copy)
- **Headline block**: 1 chosen headline + 2 alternates labeled as such
- **Body copy**: Full prose for each section in order
- **CTA buttons**: Shown as
