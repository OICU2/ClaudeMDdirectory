# Introduction Rewrite Variants

This skill produces 4–6 distinct rewrites of an existing article or essay introduction, each using a different hook style, so the writer can choose the approach that best fits their voice and audience.

## Workflow

1. **Extract the core argument**: Read the provided introduction and identify the central thesis, topic, and intended audience before writing anything.
2. **Identify what to preserve**: Note any key facts, names, or claims from the original that must appear in the rewrites.
3. **Select hook styles**: Apply each of the following styles, producing one variant per style:
   - **Anecdote**: Open with a brief, concrete scene or story (real or illustrative) that embodies the article's central tension.
   - **Statistic or data point**: Lead with a surprising or counterintuitive number that makes the stakes immediately tangible.
   - **Provocative question**: Pose a single sharp question the reader cannot immediately answer but wants to.
   - **Bold declarative claim**: State the thesis or a controversial position directly and without hedging.
   - **Contrast or paradox**: Open by placing two opposing ideas side by side to create tension.
   - **Vivid scene-setting**: Ground the reader in a specific time, place, or sensory detail before zooming out to the argument.
4. **Match tone to context**: If the original is academic, keep variants formal. If conversational, keep them accessible. Do not shift register without flagging it.
5. **Keep length consistent**: Each variant should match the approximate length of the original introduction (±20%).
6. **Label clearly**: Prefix each variant with its hook style name so the writer can scan quickly.

## Output Format

Produce each variant as a clearly labeled block in this structure:

```
### Variant 1 — [Hook Style Name]
[Rewritten introduction text]

### Variant 2 — [Hook Style Name]
[Rewritten introduction text]
```

- Minimum 4 variants, maximum 6
- After all variants, include a single short section titled **"Quick Comparison"** — one sentence per variant summarizing its strategic effect (e.g., "Builds emotional identification before introducing the argument")
- No commentary between variants; keep the output scannable
- Do not restate the original introduction unless the user explicitly asks for it as a reference
