# Microcopy Writing

This skill generates polished UI microcopy for buttons, tooltips, error messages, empty states, placeholders, and onboarding flows using UX writing best practices.

## Microcopy Writing Process

1. **Identify the element type** — Determine what is being written: button label, tooltip, error message, empty state, placeholder, success message, onboarding step, or modal copy.

2. **Extract context** — Note the product, user action, emotional state of the user at that moment, and the desired outcome. Ask if not provided.

3. **Apply element-specific rules:**
   - **Buttons:** Use verb-first, action-oriented labels (2–4 words max). Avoid "Submit" or "Click here." Prefer "Save changes," "Start free trial," "Send message."
   - **Tooltips:** One sentence. Explain what the element does, not what it is. Start with a verb when possible.
   - **Error messages:** Say what went wrong, why (if helpful), and what to do next. Never blame the user. Avoid jargon or error codes in user-facing text.
   - **Empty states:** Explain why it's empty, encourage action, include a CTA. Make it human, not clinical.
   - **Placeholders:** Give an example value, not a label. Use faded text convention (e.g., "e.g. jane@company.com").
   - **Onboarding text:** One idea per screen. Lead with benefit, not feature. Use "you" language.
   - **Success/confirmation:** Confirm what happened, set next expectations, keep it brief.

4. **Apply tone calibration** — Match the product's voice (professional, friendly, playful, etc.). If no voice is specified, default to clear, warm, and human.

5. **Write multiple variants** — Produce 2–3 options per element so the team can choose or combine.

6. **Self-check each string:**
   - Is it under the character/word limit for the context?
   - Does it assume knowledge the user may not have?
   - Is it scannable in under 2 seconds?
   - Does it avoid passive voice, jargon, and filler words?

## Output Format

For each microcopy element, produce:

```
## [Element Type] — [Context/Screen Name]

**Option A:** [copy]
**Option B:** [copy]
**Option C:** [copy] *(optional)*

**Notes:** [1–2 sentences on tone choices, character counts, or usage guidance if relevant]
```

- List all elements requested, each as its own section
- Keep notes brief and practical — only include if they add real value
- If multiple elements are requested at once (e.g., full onboarding flow), group by screen or user journey step
- Do not include placeholder brackets in final copy — all output must be production-ready
