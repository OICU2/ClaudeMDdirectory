# Byline Tone Matching

This skill analyzes writing samples from a target author to extract measurable style patterns and applies them to produce new content that authentically mirrors their voice, rhythm, and vocabulary.

## Style Extraction Workflow

**Step 1: Collect Writing Samples**
- Request 3–5 writing samples from the user (minimum 300 words each)
- If unavailable, ask for public URLs or paste-in excerpts
- Identify the content type (blog posts, essays, social copy, emails, etc.)

**Step 2: Analyze Core Style Dimensions**
Extract and document each of the following explicitly:

- **Sentence rhythm**: Average sentence length, variation pattern (e.g., short-long-short bursts vs. uniform medium), use of fragments
- **Paragraph structure**: Length norms, how paragraphs open and close, transitions style
- **Vocabulary register**: Formal vs. casual, technical density, jargon frequency, preferred word tiers (Latinate vs. Anglo-Saxon)
- **Punctuation habits**: Em dash usage, comma frequency, semicolons, ellipses, serial comma preference
- **Tone markers**: Humor style, directness level, hedging language, use of rhetorical questions
- **First-person patterns**: How (or whether) the author inserts themselves — anecdotes, "I" frequency, opinion framing
- **Signature phrases or constructions**: Recurring idioms, pet words, sentence openers they favor
- **What they avoid**: Passive voice, certain clichés, overly academic phrasing, etc.

**Step 3: Build a Style Profile**
Summarize findings in a structured style card (see Output Format).

**Step 4: Generate Matched Content**
- Apply the style profile to the user's requested topic
- Write a first draft using only constructions, rhythms, and vocabulary patterns identified in the profile
- Do not introduce stylistic elements absent from the samples

**Step 5: Self-Audit**
Before delivering, check the draft against the profile:
- Flag any sentence that breaks the established rhythm pattern
- Confirm vocabulary register is consistent throughout
- Verify tone markers appear at appropriate density
- Revise any deviations before output

## Output Format

Deliver two blocks:

**Block 1 — Style Profile Card**
```
STYLE PROFILE: [Author Name / Brand]
─────────────────────────────────────
Sentence rhythm:       [description + example]
Paragraph length:      [description]
Vocabulary register:   [description + 5 characteristic word choices]
Punctuation habits:    [list specific tendencies]
Tone:                  [2–3 adjectives + evidence]
First-person use:      [frequency + pattern]
Signature constructions: [3–5 examples lifted from samples]
Avoids:                [list]
─────────────────────────────────────
```

**Block 2 — Generated Content**
- Full draft at the requested length and format
- No meta-commentary inside the content
- If uncertainty exists about a style choice, add a bracketed note after the draft (e.g., `[Note: used em dash here — samples were ambiguous on this; adjust if needed]`)
