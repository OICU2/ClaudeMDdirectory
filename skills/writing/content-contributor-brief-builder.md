# Content Contributor Brief Builder

This skill generates a structured, publication-ready contributor brief that gives writers everything they need to produce on-target content without back-and-forth.

## Brief Construction Workflow

1. **Extract core inputs** — Identify the article topic, target publication or platform, and intended audience from the user's request. Ask clarifying questions if any of these are missing before proceeding.

2. **Define the audience profile** — Write a 2–3 sentence audience description covering demographics, knowledge level, pain points, and what they expect to gain from the piece.

3. **Establish the editorial angle** — State the specific point of view, argument, or narrative hook the piece must take. Distinguish it from generic coverage of the topic (e.g., not "how to use AI" but "why mid-market ops teams specifically resist AI adoption and how to overcome it").

4. **Set structural requirements** — Specify the required sections or headers, approximate word count per section, and total word count target. Include whether a lede, subheads, pull quotes, or a CTA are required.

5. **List required sources and research** — Provide 3–5 mandatory source types (e.g., peer-reviewed study, industry report, named expert quote, first-person case study). Name specific sources or publications if known.

6. **Apply editorial guidelines** — State tone (formal/conversational/authoritative), POV (first/second/third person), reading level target, banned phrases or topics, SEO keyword requirements if applicable, and house style notes.

7. **Add deliverable and deadline specs** — Specify file format, submission method, revision rounds included, and placeholder deadline if the user provides one.

## Output Format

Produce a single formatted contributor brief using the following structure:

```
CONTRIBUTOR BRIEF
-----------------
Article Title (or working title):
Assigned To: [Writer name or "TBD"]
Due Date: [Date or TBD]
Publication/Platform:

AUDIENCE
[2–3 sentences describing the target reader]

EDITORIAL ANGLE
[1–2 sentences: the specific argument or hook]

STRUCTURE & WORD COUNT
- Total target: [X words]
- Required sections:
  1. [Section name] (~X words)
  2. [Section name] (~X words)
  [continue as needed]
- Required elements: [lede / subheads / CTA / pull quotes — list what applies]

REQUIRED SOURCES & RESEARCH
- [Source type 1 — specific description]
- [Source type 2]
- [Source type 3]
[Add up to 5]

TONE & STYLE GUIDELINES
- Tone: [e.g., authoritative but accessible]
- POV: [first/second/third person]
- Reading level: [e.g., 8th grade / professional]
- Avoid: [phrases, topics, or approaches to exclude]
- SEO keywords: [primary keyword, secondary keywords if applicable]
- House style notes: [AP Style / Oxford comma / brand voice specifics]

DELIVERABLES
- Format: [Google Doc / Word / Markdown]
- Submission: [email / shared drive / platform]
- Revisions included: [number]
```

The brief must be immediately usable by a writer with no additional context. Flag any gaps in the user's input that would leave sections incomplete.
