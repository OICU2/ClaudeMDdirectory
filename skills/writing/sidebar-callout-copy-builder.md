# Sidebar & Callout Copy Builder

This skill drafts callout boxes, tip blocks, warnings, notes, and sidebars that add contextual value alongside body copy without echoing what the main text already says.

## Callout Writing Workflow

1. **Identify the callout type** before writing. Common types and their purposes:
   - **Tip / Pro Tip** — actionable shortcut or enhancement the reader can apply immediately
   - **Warning / Caution** — consequence-focused; flags what goes wrong if the reader skips something
   - **Note / FYI** — supplementary context that enriches but isn't essential to the main flow
   - **Important** — non-optional information the reader must not miss
   - **Example** — concrete illustration that reinforces a concept without restating the explanation
   - **Definition** — clarifies jargon or a key term introduced nearby
   - **Pull Quote** — a resonant phrase lifted or paraphrased from the body copy to anchor a section visually

2. **Read the surrounding body copy** (or ask for it). Identify what the main text already covers. The callout must not repeat that content — it must extend, contrast, or concretize it.

3. **Apply the callout's job** to the specific subject matter:
   - Tips: lead with the action, then the benefit ("Use X to achieve Y in half the time.")
   - Warnings: lead with the risk, then the fix ("Skipping X can cause Y — always Z first.")
   - Notes: lead with the new angle, keep it to 1–2 sentences
   - Definitions: one crisp sentence, plain language, no circular phrasing
   - Pull quotes: 10–25 words, rhythmically strong, no subject-verb awkwardness

4. **Check for duplication.** If the callout restates a sentence already in the body, rewrite it to add a fact, example, edge case, or contrasting perspective instead.

5. **Match the voice and register** of the surrounding copy. A formal whitepaper gets a formal callout; a casual blog gets a conversational one.

6. **Keep it short.** Callouts that run longer than 60 words lose scannability. Trim ruthlessly.

## Output Format

Produce the following for each callout requested:

```
[CALLOUT TYPE LABEL]
Headline (optional, 4–8 words): ...
Body: ...
```

- **Label** — the callout type in brackets (e.g., `[TIP]`, `[WARNING]`, `[NOTE]`)
- **Headline** — include only if the format typically uses one (tips, warnings, important blocks); omit for pull quotes and short notes
- **Body** — 1–3 sentences, 15–60 words, no bullet lists inside the callout unless the request explicitly asks for a checklist-style block
- If multiple callouts are requested, separate each with a blank line and a horizontal rule (`---`)
- If the surrounding copy was provided, append a one-line note: *"Does not duplicate: [specific point avoided]"* so the requester can verify at a glance
