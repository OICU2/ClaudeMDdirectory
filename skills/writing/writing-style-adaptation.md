# Writing Style Adaptation

This skill rewrites or generates content by calibrating tone, vocabulary, sentence complexity, and formality to precisely match a target audience or context.

## Style Adaptation Workflow

1. **Identify the target audience** — Determine who will read this: age group, expertise level, relationship (colleague, customer, executive, child), and cultural context.

2. **Identify the target context** — Determine where the content lives: email, social post, technical doc, marketing copy, academic paper, chat message, speech.

3. **Diagnose the current style** — Note what needs to change: vocabulary complexity, sentence length, passive vs. active voice, jargon density, formality level, emotional warmth.

4. **Apply style parameters by audience type:**
   - **Expert/Technical**: Use domain jargon freely, dense information, assume prior knowledge, minimal hand-holding
   - **General/Consumer**: Plain language, short sentences, define technical terms, concrete examples
   - **Executive**: Lead with outcomes, bullet-heavy, no fluff, quantify impact
   - **Beginner/Child**: Simple words (≤2 syllables where possible), analogies to familiar concepts, encouraging tone
   - **Peer/Colleague**: Conversational, moderate formality, shared context assumed
   - **Formal/Legal**: Precise language, no contractions, complete sentences, hedged claims

5. **Preserve core meaning** — Every key idea, fact, and instruction from the original must survive the rewrite intact.

6. **Check consistency** — Scan the full output for tone drift; every sentence should feel like it came from the same voice.

## Output Format

- Deliver the rewritten content directly — no preamble
- If rewriting existing content: provide the full adapted version, not just changed sections
- After the rewrite, include a brief **Style Notes** block (2–4 bullet points) identifying the specific changes made: e.g., "Replaced passive constructions with active voice," "Reduced average sentence length from ~22 to ~12 words," "Removed 6 industry acronyms"
- If the request is ambiguous about audience, ask one clarifying question before rewriting: "Who is the primary reader for this?"
- Length of output should match length of input unless the style shift inherently requires expansion or compression
