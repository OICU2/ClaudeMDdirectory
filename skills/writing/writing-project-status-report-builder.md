# Writing Project Status Report Builder

Generates a structured, scannable status report for any writing project by organizing completed work, active tasks, blockers, and upcoming deadlines into a clear snapshot document.

## Workflow

1. **Gather project context** — Ask for or extract: project name, type (book, article series, screenplay, report, etc.), total scope (word count, chapter count, or piece count), and target completion date.
2. **Inventory completed work** — List every draft, section, or piece that is fully written. Note version number or draft stage (first draft, revised draft, final) for each item.
3. **Document outstanding revisions** — Identify all sections or pieces that exist but require further work. Specify what kind of revision is needed (structural, copy editing, fact-checking, formatting).
4. **Log pending approvals and reviews** — List any items submitted to editors, clients, or reviewers that are awaiting feedback or sign-off. Include who holds each item and how long it has been waiting.
5. **Flag blockers** — Note any items that cannot move forward due to missing information, unavailable collaborators, or unresolved decisions.
6. **Map upcoming milestones** — List the next 3–5 deadlines or deliverables in chronological order with due dates.
7. **Calculate overall completion percentage** — Estimate progress as a percentage based on completed pieces versus total project scope.

## Output Format

Produce a titled status report with the following sections in this order:

```
# [Project Name] — Writing Status Report
**Date:** [Today's date]
**Overall Progress:** [X]% complete

---

## ✅ Completed Drafts
- [Item name] — [Draft stage] ([word/page count if known])

## ✍️ Outstanding Revisions
- [Item name] — [Revision type needed]

## ⏳ Pending Approvals
- [Item name] — Submitted to [person/team] on [date], awaiting [feedback/sign-off]

## 🚧 Blockers
- [Item name] — [Description of blocker and what is needed to resolve it]

## 📅 Upcoming Milestones
| Milestone | Due Date | Owner |
|-----------|----------|-------|
| [Item]    | [Date]   | [Name]|

## Summary
[2–3 sentence plain-language summary of where the project stands, the biggest risk or priority, and the recommended next action.]
```

Keep the report skimmable. Use bullet points, not paragraphs, in all sections except the Summary. If information for a section is unavailable, write "Nothing to report" rather than omitting the section.
