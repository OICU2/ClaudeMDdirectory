# Pilot Program Scoping Template

This skill generates a complete, structured pilot program proposal document that enterprise sales teams can use to align with prospects on scope, success criteria, and expansion paths.

## Pilot Scoping Workflow

1. **Gather Context First**
   - Ask for: prospect company name, industry, deal size, primary use case, key stakeholders, and desired pilot duration if not already provided
   - Identify the prospect's stated pain points and what "success" means to them informally before formalizing it

2. **Define Pilot Scope**
   - Set a specific pilot duration (recommend 30, 60, or 90 days based on complexity)
   - Define included features/modules — explicitly list what is IN scope and what is OUT of scope
   - Specify the user group: number of seats, departments, and geographic regions covered
   - Identify the single primary use case the pilot will validate

3. **Build Success Metrics**
   - Create 3–5 quantifiable KPIs tied directly to the prospect's stated pain points
   - Each metric must have: a baseline value (current state), a target value (expected improvement), and a measurement method
   - Include at least one business outcome metric (e.g., time saved, revenue influenced, cost reduced) and one adoption metric (e.g., DAU, feature utilization rate)

4. **Assign Stakeholder Responsibilities**
   - List roles on both the vendor side and the prospect side
   - For each role, define: name/title, primary responsibility during pilot, and escalation path
   - Identify the single executive sponsor on the prospect side who owns the go/no-go decision

5. **Set the Timeline with Milestones**
   - Week 1: Technical setup, onboarding, and kickoff
   - Week 2–3: Active usage and first check-in
   - Midpoint: Formal progress review against KPIs
   - Final week: Data collection and success review preparation
   - End of pilot: Executive business review (EBR) with expansion recommendation

6. **Define Expansion Triggers**
   - List 2–3 specific metric thresholds that, if met, automatically justify expansion
   - Include a named next step if triggers are met (e.g., full deployment proposal, contract expansion, additional seat order)
   - Include one contingency: what happens if metrics are partially met (e.g., extended pilot, scope adjustment)

7. **Outline Mutual Commitments**
   - Vendor commits: dedicated CSM, technical support SLA, weekly check-ins, and final ROI report
   - Prospect commits: named pilot champion, user participation minimums, data access for measurement, and attendance at EBR
