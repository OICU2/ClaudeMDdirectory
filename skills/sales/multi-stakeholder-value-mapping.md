# Multi-Stakeholder Value Mapping

This skill builds a structured value proposition matrix for every stakeholder in a buying committee, ensuring each person's specific priorities, pain points, and success metrics are addressed with targeted messaging.

## Stakeholder Mapping Workflow

**Step 1: Identify the Buying Committee**
Ask for or infer the list of stakeholder roles involved in the deal. Common roles include:
- Economic Buyer (CFO, VP Finance)
- Technical Buyer (CTO, IT Director, Security)
- End User / Champion (Department Head, Practitioner)
- Legal / Compliance
- Procurement
- Executive Sponsor (CEO, COO)

If roles are not provided, prompt the user: "Who are the key stakeholders or roles involved in this decision?"

**Step 2: Gather Context**
Collect the following before mapping:
- Product or solution being sold
- Industry and company size of the prospect
- Any known pain points or objections per stakeholder
- Deal stage and urgency signals

**Step 3: Map Each Stakeholder**
For every identified stakeholder, define:
1. **Primary Motivation** — What outcome does this role care most about? (cost reduction, risk mitigation, efficiency, innovation, compliance, career safety)
2. **Key Pain Points** — What problems keep them up at night in their role?
3. **Success Metric** — How does this stakeholder measure a win? (ROI percentage, hours saved, incidents reduced, adoption rate)
4. **Value Proposition** — One to two sentences tailored specifically to their lens
5. **Likely Objection** — The most predictable pushback from this role
6. **Objection Response** — A concise reframe or counter-argument

**Step 4: Identify Alignment and Conflicts**
- Flag where stakeholder interests align (use as coalition-building opportunities)
- Flag where stakeholder interests conflict (e.g., IT wants control; end users want flexibility) and suggest bridging language

**Step 5: Recommend Engagement Sequence**
Suggest the order in which to engage stakeholders based on influence, urgency, and deal risk.

## Output Format

Produce a stakeholder value map with the following structure for each role:

---

### [Stakeholder Role] — [Name if known]

| Field | Content |
|---|---|
| Primary Motivation | [1 sentence] |
| Key Pain Points | [2–3 bullet points] |
| Success Metric | [Specific, measurable outcome] |
| Tailored Value Proposition | [1–2 sentences spoken in their language] |
| Likely Objection | [1 sentence] |
| Objection Response | [2–3 sentences] |

---

After all stakeholders are mapped, include:

**Alignment Opportunities:**
- [Bullet list of shared interests across roles]

**Conflict Zones:**
- [Bullet list of competing priorities with suggested bridging language]

**Recommended Engagement Sequence:**
1. [Role] — [Reason]
2. [Role] — [Reason]
(continue for all roles)

Length: One table block per stakeholder. Total output should be comprehensive but scannable — no paragraph walls. Use plain language matched to each stakeholder's professional vocabulary.
