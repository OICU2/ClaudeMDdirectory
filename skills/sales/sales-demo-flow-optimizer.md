# Sales Demo Flow Optimizer

Reorders and rewrites a demo script so each feature walkthrough directly addresses a pain point from the discovery call, maximizing relevance and deal momentum.

## Optimization Workflow

1. **Extract Discovery Pain Points**
   - Ask the user to provide discovery call notes, a summary, or a list of stated pain points
   - Categorize each pain point by urgency (critical blocker, moderate friction, nice-to-fix)
   - Identify the buyer's language and exact phrasing they used to describe each problem

2. **Audit the Existing Demo Script**
   - Ask for the current demo script or feature walkthrough list
   - Map every feature or demo section to a potential pain point it solves
   - Flag any demo sections that address no discovered pain point — these are deprioritized or cut

3. **Build the Pain-to-Feature Map**
   - Create an explicit mapping: [Pain Point] → [Feature/Demo Moment] → [Proof of Resolution]
   - Ensure the top 1–2 critical blockers are addressed within the first one-third of the demo
   - Group related features that solve the same pain point into a single narrative arc

4. **Reorder the Demo Flow**
   - Open with the buyer's most critical pain point — not a product overview
   - Sequence remaining sections from highest to lowest pain point urgency
   - Insert transition lines between sections that reference the previously solved pain and set up the next
   - Move any product strengths that have no matching pain point to the appendix or cut entirely

5. **Rewrite Section Introductions**
   - Each demo section must open with the buyer's own language: *"You mentioned that [pain point]..."*
   - Follow immediately with the feature demonstration
   - Close each section with a proof statement or outcome: *"This is how [customer] reduced X by Y%"*

6. **Add Discovery Callbacks**
   - Insert at least one callback to discovery notes per major section
   - Add a closing summary that recaps each pain point and how the demo addressed it

## Output Format

Produce a restructured demo script with the following sections:

**Pain Point Priority Map**
A ranked table: | Priority | Pain Point (buyer's words) | Feature/Section | Outcome Proof |

**Optimized Demo Flow**
Numbered sections in recommended run order, each containing:
- Section title
- Opening line using buyer's language
- Feature walkthrough notes (2–4 bullet points)
- Transition line to next section
- Suggested time allocation (minutes)

**Cut or Deprioritized Content**
A short list of original demo sections removed or moved to appendix, with a one-line reason for each.

**Discovery Callback Cheat Sheet**
5–8 one-liner phrases the seller can use verbatim during the demo to reference discovery, formatted as a quick-reference list.

Total output length: scaled to demo complexity, typically 400–800 words for the restructured flow.
