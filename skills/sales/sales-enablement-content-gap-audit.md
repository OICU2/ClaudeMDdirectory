# Sales Enablement Content Gap Audit

This skill audits existing sales collateral against each active deal stage to pinpoint exactly which content gaps are causing pipeline stalls, then outputs a prioritized list of content to create.

## Audit Workflow

### Step 1: Map the Deal Stages
- Ask the user to list their pipeline stages (e.g., Prospecting, Discovery, Demo, Proposal, Legal Review, Closed Won/Lost).
- If they use a CRM, ask them to paste stage names and current deal counts per stage.
- Identify which stages have the highest drop-off or longest average age.

### Step 2: Inventory Existing Collateral
- Ask the user to list or paste their current sales assets (one-pagers, case studies, battle cards, ROI calculators, objection guides, demo decks, proposal templates, security docs, etc.).
- Tag each asset to the stage(s) where it is typically used.
- Note format (PDF, deck, video, live demo, email template) for each asset.

### Step 3: Identify Stage-to-Content Gaps
For each deal stage, evaluate:
- **Awareness gap**: Is there content to create urgency or educate on the problem?
- **Proof gap**: Is there social proof (case studies, references, G2 reviews) appropriate to the buyer's industry or persona?
- **Objection gap**: Are common objections at this stage addressed in existing material?
- **Next-step gap**: Is there a clear CTA asset to advance the deal to the next stage?
- **Stakeholder gap**: Is there content tailored to each buying committee member who typically enters at this stage (champion, economic buyer, legal, IT)?

### Step 4: Cross-Reference Against Stuck Deals
- Ask: "Which stage are your deals most commonly stalling in, and what reason is given when deals go dark?"
- Map stated stall reasons directly to missing content types (e.g., "waiting on security review" → no security one-pager or trust packet exists).

### Step 5: Score and Prioritize Gaps
Rank each identified gap by:
1. **Pipeline impact**: How many active deals are affected?
2. **Stage criticality**: Is this a late-stage gap (higher revenue risk)?
3. **Creation effort**: Can this be produced quickly (email template) vs. requires significant effort (video)?

## Output Format

Produce a structured audit report with the following sections:

**1. Stage Coverage Heatmap**
A table with deal stages as rows and content types as columns (Awareness, Proof, Objection Handling, Next-Step CTA, Stakeholder-Specific). Mark each cell: ✅ Exists | ⚠️ Partial | ❌ Missing.

**2. Top 5 Priority Content Gaps**
For each gap:
- Stage where it applies
- Content type needed
- Why it's causing stalls (specific deal behavior or objection)
- Recommended format and estimated creation effort (Low / Medium / High)
- Suggested owner (marketing, sales ops, product marketing, AE)

**3. Quick Wins (Buildable in < 1 Week)**
A bulleted list of 3–5 assets that can be created immediately with existing materials (repurposed emails, pulled quotes, reformatted decks).

**4. 30-Day Content Roadmap**
A prioritized backlog of all identified gaps ordered by pipeline impact, with suggested sequencing for the content team.

Keep the tone direct and tactical. No generic advice — every recommendation must tie back to a specific stage, a specific gap type, and an observed stall pattern.
