# Competitive Objection Reframing

This skill generates structured response scripts that acknowledge a prospect's competitive objection, then pivot the conversation to evaluation criteria where your solution has a clear, provable edge.

## Reframing Workflow

1. **Extract the objection details** — Identify the competitor named, the specific objection type (price, features, incumbent inertia, past failure, brand preference), and any context about the prospect's role or priorities.

2. **Acknowledge without conceding** — Open the script with a one-sentence validation that confirms you heard the concern without agreeing it's a disqualifying factor. Avoid dismissive language.

3. **Identify the reframe axis** — Choose the criteria shift based on objection type:
   - *Price objection* → shift to total cost of ownership, hidden fees, or ROI over 12–24 months
   - *Feature objection* → shift to outcomes delivered vs. features listed, or depth vs. breadth
   - *Incumbent inertia* → shift to switching cost of staying (risk, missed growth, technical debt)
   - *Past failure* → shift to what specifically failed and how this solution addresses that root cause
   - *Brand preference* → shift to fit-for-size, support model, or customer success track record

4. **Build the pivot statement** — Write 2–3 sentences that introduce the new evaluation lens using the prospect's own stated priorities as the anchor. Reference a concrete metric, case study type, or proof point placeholder where evidence should be inserted.

5. **Write the closing question** — End with an open-ended question that invites the prospect to evaluate on the new criteria, not the original objection terms.

6. **Flag assumption risks** — Note any place in the script where the reframe only holds if a specific product advantage is real. Mark these with `[VERIFY: ...]` so the rep can confirm before using.

## Output Format

Produce the following structure for each objection submitted:

**Objection Received:** [restate the objection in one line]

**Reframe Strategy:** [one sentence naming the criteria shift and why it applies]

**Response Script:**
> [Full word-for-word script in the rep's voice, 4–7 sentences, ready to say aloud or paste into email. Use `[PROOF POINT]` as a placeholder where a stat, case study, or customer name should be inserted.]

**Closing Question:**
> [One open-ended question to end the exchange]

**Flags:**
- `[VERIFY: ...]` items listed as bullets — one per assumption that must be true for the reframe to hold

If multiple objections are provided at once, repeat this structure for each one. Keep each script tight and conversational — no jargon, no corporate filler.
