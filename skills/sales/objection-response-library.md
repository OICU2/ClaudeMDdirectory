# Objection Response Library

This skill builds a structured, role-specific repository of sales objections paired with multi-step response frameworks, helping reps navigate pushback confidently during live calls.

## Workflow

1. **Identify scope** — Ask (or infer) the sales role, deal stage, and product/service category. A mid-market AE closing SaaS deals needs different frameworks than an SDR running cold outreach.

2. **Extract or generate objections** — If the user provides objections, work from those. Otherwise, generate the 8–12 most common objections for the specified context. Organize by category:
   - Price / Budget ("It's too expensive," "We don't have budget right now")
   - Timing ("We're not ready to move yet," "Call me next quarter")
   - Competition ("We're already using a competitor," "We're evaluating other options")
   - Authority ("I need to loop in my boss," "This isn't my decision")
   - Value / Fit ("I don't see how this applies to us," "We can build this internally")
   - Trust / Risk ("We've never heard of you," "What if it doesn't work?")

3. **Build each response framework** using this structure per objection:
   - **Acknowledge** — Validate the concern without capitulating (1 sentence)
   - **Clarify** — Ask a probing question to uncover the real objection beneath the surface (1 question)
   - **Reframe** — Shift perspective using a value anchor, relevant proof point, or ROI logic (2–3 sentences)
   - **Advance** — Propose a clear next step that moves the deal forward (1 sentence)

4. **Tailor language** to the rep's persona — use the vocabulary, tone, and scenarios relevant to their industry and buyer profile. Avoid generic scripts; make responses sound human and conversational.

5. **Flag escalation triggers** — For each objection, note one signal that indicates the objection is a dealbreaker vs. a negotiating tactic, and advise how to respond differently in each case.

## Output Format

Produce a structured library in this format for each objection:

---
**Objection:** [Exact or near-exact language the prospect uses]
**Category:** [Price / Timing / Competition / Authority / Value / Trust]
**Dealbreaker signal:** [One specific indicator this is real vs. a tactic]

**Response Framework:**
- **Acknowledge:** [1 sentence validating the concern]
- **Clarify:** [1 probing question]
- **Reframe:** [2–3 sentences repositioning value]
- **Advance:** [1 sentence proposing next step]

**If dealbreaker:** [1–2 sentences on how to handle gracefully and preserve the relationship]
---

Deliver all objections in one continuous document. Use headers to separate categories. Total length should be scannable in under 5 minutes — aim for 8–12 objections, each framework fitting within 10 lines.
