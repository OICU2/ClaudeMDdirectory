# Mutual Action Plan Creation

This skill builds a structured, collaborative milestone document that assigns tasks, owners, and deadlines to both the buyer and seller, reducing deal slippage by creating shared accountability.

## Mutual Action Plan Workflow

1. **Gather deal context** — Ask for or extract: prospect company name, deal stage, target close date, key stakeholders on both sides, and any known blockers or pending requirements (e.g., security review, legal approval, budget sign-off).

2. **Identify all required milestones** — Map every action that must occur before the deal closes. Group into categories:
   - **Evaluation/Technical** (demos, POCs, security questionnaires, integration reviews)
   - **Business/Financial** (ROI review, budget approval, executive sponsor alignment)
   - **Legal/Procurement** (contract redlines, MSA, NDA, vendor registration)
   - **Decision & Signature** (final presentation, approvals, DocuSign)

3. **Assign ownership** — Every task gets one clear owner: either a named buyer stakeholder or a named seller team member. No orphaned tasks.

4. **Set deadlines** — Work backward from the target close date. Assign a specific due date (not "ASAP") to each task. Flag any tasks that are on the critical path.

5. **Define success criteria** — For each milestone, note what "done" looks like (e.g., "Security review complete" = written approval from buyer's CISO).

6. **Add a shared goal statement** — Open the MAP with one sentence both parties agree on: the mutual objective and target go-live or close date.

7. **Flag risks** — Note any dependencies or blockers that could delay the timeline, and suggest mitigation steps.

## Output Format

Produce a MAP in this structure:

---

**Mutual Action Plan**
**Prospect:** [Company Name] | **Seller:** [Your Company]
**Target Close Date:** [Date] | **Go-Live / Start Date:** [Date]

**Shared Goal:** [One sentence describing the mutual objective and desired outcome.]

---

| # | Milestone | Task Description | Owner | Due Date | Status | Notes |
|---|-----------|-----------------|-------|----------|--------|-------|
| 1 | [Category] | [Specific action] | [Name, Company] | [MM/DD] | Not Started | [Dependency or blocker] |
| 2 | ... | ... | ... | ... | ... | ... |

---

**Critical Path Items:** [Bullet list of tasks that, if delayed, push the close date]

**Open Risks:**
- [Risk 1] → Mitigation: [Action]
- [Risk 2] → Mitigation: [Action]

**Next Sync:** [Suggested date and agenda for next buyer-seller check-in]

---

The table should contain 8–15 rows for a typical mid-market deal. Adjust scope for enterprise (more rows, more detail) or SMB (fewer rows, faster cadence). Use plain Markdown so it can be pasted into email, Google Docs, or a CRM.
