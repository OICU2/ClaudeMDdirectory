# Sales Call Talk Track Builder

Generates a complete, stage-specific sales call talk track with industry-tailored language, discovery questions, transition phrases, and closing prompts that a rep can use verbatim or adapt on the fly.

## Talk Track Construction Workflow

1. **Identify the call stage** — Determine which stage the call falls into: cold outreach, discovery, demo/presentation, negotiation, or close. If unclear, ask the user before proceeding.

2. **Gather prospect context** — Collect the following before building:
   - Prospect's industry (e.g., healthcare, SaaS, manufacturing, retail)
   - Prospect's role/title (e.g., VP of Sales, CTO, Procurement Manager)
   - Known pain points or trigger events (e.g., recent funding, hiring spike, churn problem)
   - Product or solution being sold
   - Any previous interactions or objections raised

3. **Map the talk track to the call stage** — Apply the correct structure per stage:
   - **Cold/Intro call**: Permission opener → one-sentence value prop → single qualifying question → soft next step
   - **Discovery call**: Rapport opener → situational questions → problem questions → implication questions → need-payoff questions (SPIN structure) → summary of pain → proposed next step
   - **Demo/Presentation**: Recap of discovery pain → agenda confirmation → feature-to-pain mapping → proof point per pain → trial close after each section
   - **Negotiation**: Acknowledge concern → anchor to value already established → trade, don't give → conditional close
   - **Closing call**: Recap ROI/impact → address final objections → ask for the business directly → silence → next step regardless of outcome

4. **Inject industry-specific language** — Replace generic phrases with terminology native to the prospect's industry. Use their likely KPIs, regulatory concerns, and common workflows (e.g., for healthcare: "patient outcomes," "compliance burden," "EHR integration"; for SaaS: "churn," "activation rate," "time-to-value").

5. **Write transition phrases** — Include at least 3 explicit verbal transitions between talk track sections (e.g., "Based on what you just shared…", "That actually leads me to something most [industry] teams tell us…", "Before I show you X, can I ask one more thing?").

6. **Add objection inserts** — For the identified stage, include 2–3 likely objections with a talk track response for each. Format as: **Objection → Bridge → Reframe → Question to re-engage**.

7. **Close with a next-step prompt** — End every talk track with a specific, assumptive next step tied to the stage (e.g., "Based on what we covered, does Thursday or Friday work better for a 30-minute demo with your team?").

## Output Format

Produce a structured talk track document with the following sections, clearly labeled:

---

**Call Stage:** [Identified stage]
**Prospect Profile:** [Industry | Role | Known Pain | Product Being Sold]

---

### Opening
[Verbatim or near-verbatim opening lines, 3–5 sentences]

### Discovery / Qualification Questions
[Numbered list of 5–8 questions, ordered logically, with a one-line note on what each question is designed to uncover]

### Transition Phrases
[Bulleted list of 3–5 phrases to move between sections naturally]

[Table or bulleted list: Pain Point → Relevant Feature → Proof Point]
