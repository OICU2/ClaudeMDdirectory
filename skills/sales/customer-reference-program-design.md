# Customer Reference Program Design

This skill builds an end-to-end framework for identifying, recruiting, and activating satisfied customers as sales references, producing a ready-to-use program structure for supporting late-stage deals and POC requests.

## Program Design Workflow

### 1. Define Reference Tiers
Create three distinct reference levels based on commitment required:
- **Tier 1 — Light Touch**: Allows logo use, written quote, case study participation
- **Tier 2 — Active Reference**: Takes peer calls, joins analyst interviews, participates in webinars
- **Tier 3 — Champion**: Speaks at events, joins advisory board, co-authors content, joins prospect calls or site visits

### 2. Identify Candidate Criteria
Score existing customers against these signals:
- NPS score ≥ 8 or CSAT ≥ 85%
- Active product usage (define a usage threshold relevant to the product)
- Measurable ROI achieved (quantified outcomes documented)
- Positive support history (low escalation rate)
- Relationship health with CSM rated "green"
- No active renewal risk or open commercial disputes

### 3. Build the Recruitment Motion
Define a clear ask sequence:
1. CSM or AE identifies candidate and flags in CRM
2. CSM sends a warm, personalized outreach email explaining the program and benefits
3. Offer a reference benefit menu (early access, executive briefings, recognition, event invitations, co-marketing)
4. Conduct a 15-minute onboarding call to confirm comfort level, topics, and boundaries
5. Document approved use cases, personas they will speak to, and any off-limits topics
6. Log reference profile in a dedicated reference management tool or CRM record

### 4. Build the Reference Profile
For each enrolled reference, capture:
- Company name, industry, size, and geography
- Primary contact name, title, and communication preference
- Use case and product modules in use
- Key outcomes achieved (specific metrics)
- Approved reference activities (Tier 1/2/3)
- Personas and deal stages they are best suited for
- Availability cadence and blackout periods
- Last activation date (to prevent overuse)

### 5. Define the Activation Process
Create a standard request and fulfillment workflow:
1. Sales rep submits a reference request via a form or CRM task (deal stage, prospect profile, ask type)
2. Reference program manager or CSM matches request to best-fit reference within 24–48 hours
3. Reference is briefed on the prospect context before any interaction
4. Post-call debrief collected from both the reference and the prospect
5. Reference is thanked and their activity is logged

### 6. Set Program Health Metrics
Track these KPIs quarterly:
- Total enrolled references by tier
- Reference request fulfillment rate and time-to-match
- Win rate for deals that used a reference vs. those that did not
- Reference utilization rate (flag contacts used more than 2x per quarter)
- Reference churn rate (contacts who opt out)
- New reference recruitment rate

### 7. Protect Reference Relationships
Establish hard rules:
- No reference used more than twice per quarter without explicit consent
- Sales reps cannot contact references directly without program manager approval
- Every reference interaction ends with a thank-you and a value-add (event invite, product update, swag)
- References are reviewed annually to confirm continued eligibility
