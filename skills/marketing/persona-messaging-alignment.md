# Persona Messaging Alignment

This skill builds a structured messaging framework for each customer persona so every campaign delivers the right message, in the right format, through the right channel — activating when teams need to move beyond generic marketing.

## Persona Mapping Workflow

**Step 1: Extract or Define Personas**
- Ask the user to list their key customer personas (2–6 is ideal)
- For each persona, collect or infer: role/identity, primary goal, biggest pain point, objection to buying, and current awareness level (unaware → solution-aware → brand-aware)
- If the user has no defined personas, prompt them with: industry, product type, and who their best customers are, then generate draft personas

**Step 2: Identify Messaging Pillars Per Persona**
- Define 3 core messaging pillars for each persona (what you want them to think, feel, and do)
- Anchor each pillar to the persona's specific pain point or desire — never use generic brand copy
- Flag where two personas share a pillar vs. where they diverge — divergence points require separate assets

**Step 3: Map Channel Preferences**
- Assign 2–3 primary channels per persona based on their role and behavior (e.g., executives → LinkedIn + email digest; developers → GitHub + technical blog; SMB owners → Facebook + SMS)
- Note content consumption habits: skimmers vs. deep readers, video vs. text, synchronous vs. asynchronous

**Step 4: Specify Content Formats**
- For each persona + channel combination, specify the exact format: short-form social post, long-form case study, 60-second video, ROI calculator, comparison table, testimonial snippet, etc.
- Flag which formats can be repurposed across personas vs. which must be built from scratch

**Step 5: Write Sample Messaging**
- Produce 1–2 sample headline/hook variants per persona that demonstrate the messaging pillar in action
- Each sample must use the persona's language, reference their specific pain point, and avoid jargon the persona wouldn't use

## Output Format

Produce a Persona Messaging Alignment Table for each persona using this structure:

---

**Persona: [Name/Label]**
- **Profile:** [2-sentence description — who they are, what they care about]
- **Awareness Level:** [Unaware / Problem-aware / Solution-aware / Brand-aware]
- **Core Pain Point:** [1 sentence, specific]
- **Messaging Pillars:**
  1. [Pillar 1 — what they should think]
  2. [Pillar 2 — what they should feel]
  3. [Pillar 3 — what they should do]
- **Primary Channels:** [Channel 1, Channel 2, Channel 3]
- **Content Formats:** [Format per channel, e.g., "LinkedIn: thought leadership post | Email: case study digest | Webinar: live demo"]
- **Sample Headlines:**
  - "[Headline variant 1]"
  - "[Headline variant 2]"
- **Do Not Use:** [Messaging angles, words, or formats that will repel or confuse this persona]

---

Repeat the block for every persona. End with a **Campaign Divergence Summary** — a short bulleted list of where campaigns must be split by persona vs. where a single asset can serve multiple personas with minor edits.
