# Podcast Episode to Blog Repurposer

This skill transforms raw podcast transcripts into structured, publication-ready blog posts with proper headings, pull quotes, and SEO-aligned summaries that capture the episode's core value in written form.

## Workflow

1. **Ingest and analyze the transcript** — Identify the central thesis, key arguments, notable stories or examples, and any quotable moments. Note the guest name, host name, and episode topic if present.

2. **Extract core themes** — Pull out 3–5 major talking points that will become the blog post's main sections. Discard filler content (verbal tics, off-topic tangents, repeated restarts) without losing substance.

3. **Generate an SEO-aligned title** — Write a title that reflects the episode's primary keyword intent. Format: `[Specific Outcome or Insight]: [How/Why/What] [Topic]`. Avoid clickbait; prioritize clarity and search relevance.

4. **Write the introduction** — Open with a hook (a surprising stat, bold claim, or compelling question from the episode). In 2–3 sentences, establish the problem or topic and tell the reader what they will learn. End with a brief guest/host intro if applicable.

5. **Build the body sections** — Create one H2 heading per major theme. Under each heading:
   - Write 150–250 words of prose synthesizing the discussion (do not transcribe verbatim)
   - Include one direct pull quote per section (exact words from the transcript, formatted as a blockquote)
   - Add a one-sentence takeaway at the end of each section

6. **Write the conclusion** — Summarize the 3 most actionable insights in a bulleted list, then close with a 2–3 sentence paragraph that reinforces the episode's main message.

7. **Craft the SEO meta summary** — Write a 150–160 character meta description that includes the primary keyword, the guest or host name if notable, and a clear value proposition.

8. **Suggest internal linking opportunities** — Identify 2–3 topic areas where the author could link to related content (e.g., "Link to your post on [topic X] when discussing [section Y]").

## Output Format

Deliver the following in order, separated by clear labels:

**BLOG POST TITLE**
A single H1-ready headline.

**META DESCRIPTION**
One sentence, 150–160 characters, SEO-optimized.

**BLOG POST BODY**
- Introduction (150–200 words)
- 3–5 H2 sections (150–250 words each, one blockquote per section, one takeaway line)
- Conclusion with bulleted insight list + closing paragraph (100–150 words)

**PULL QUOTES (STANDALONE)**
A list of 3–5 of the strongest quotes from the transcript, formatted for use as social media or graphic overlays. Each quote under 280 characters.

**INTERNAL LINKING SUGGESTIONS**
2–3 bullet points identifying topic areas and the sections where links would fit naturally.

Total blog post length: 800–1,500 words depending on transcript length. Tone should match the podcast's register — if the episode is conversational, keep prose approachable; if technical, maintain precision.
