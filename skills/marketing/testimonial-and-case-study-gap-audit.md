# Testimonial and Case Study Gap Audit

This skill cross-references existing case studies and testimonials against a company's target verticals, use cases, and buyer personas to produce a prioritized list of missing social proof assets.

## Audit Workflow

### Step 1: Inventory Existing Assets
Collect and categorize all current case studies, testimonials, G2/Capterra reviews, and video testimonials. Tag each asset with:
- **Vertical/Industry** (e.g., fintech, healthcare, e-commerce)
- **Company size** (SMB, mid-market, enterprise)
- **Buyer persona** (e.g., CTO, VP Marketing, Operations Manager)
- **Primary use case** (e.g., cost reduction, time savings, compliance, growth)
- **Deal stage where used** (awareness, consideration, decision)
- **Asset format** (written case study, video, pull quote, stat-only)

### Step 2: Define the Coverage Matrix
Build a matrix of all target segments the company sells into:
- List every target vertical the sales team actively pursues
- List every buyer persona involved in deals (champion, economic buyer, blocker)
- List every core use case the product solves
- List every company size tier in the ICP

### Step 3: Map Assets to Matrix
Place each existing asset into the coverage matrix. Mark each cell as:
- **Covered** — at least one strong, recent asset exists
- **Weak** — asset exists but is outdated (>2 years), lacks metrics, or is low-credibility
- **Missing** — no asset exists for this combination

### Step 4: Score Gaps by Priority
Rank each gap using these criteria:
1. **Pipeline frequency** — how often does this vertical/persona/use case appear in active deals?
2. **Conversion impact** — is lack of proof causing lost deals? (check CRM loss reasons, sales team feedback)
3. **Deal size** — higher ACV segments get higher priority
4. **Competitive exposure** — does a competitor have strong proof here and you don't?

Assign each gap a priority score: **Critical / High / Medium / Low**

### Step 5: Identify Candidate Customers
For each Critical or High gap, surface 2–3 existing customers who could fill it:
- Query CRM or customer success data for customers matching the missing segment
- Flag customers with high NPS scores, expansion revenue, or active advocacy status
- Note relationship owner (CSM or AE) for outreach

### Step 6: Flag Format Gaps
Beyond topic gaps, audit for format coverage:
- Is there a video testimonial for at least the top 3 verticals?
- Are there stat-heavy, metric-driven assets for the economic buyer?
- Are there short pull quotes suitable for sales decks and email sequences?
- Are there peer-review-site quotes (G2, Capterra) for self-serve or PLG motions?

## Output Format

Produce the following deliverables in order:

**1. Coverage Matrix Table**
A markdown table with verticals as rows and persona/use case combinations as columns. Each cell contains: Covered ✅ / Weak ⚠️ / Missing ❌

**2. Prioritized Gap List**
A ranked list of gaps in this format:
```
Priority: Critical
Gap: No case study for [Healthcare / VP of Operations / Compliance Use Case]
Pipeline Exposure: 14 open deals in segment
Recommended Format: Written case study with audit-trail metrics
Candidate Customers: [Acme Health — CSM: Jane Doe], [Beta Systems — AE: John Smith]
```

**3. Quick Wins List**
Bullet list of 3–5 gaps that can be closed fastest (existing customers identified, relationship warm, story clear).
