# Product-Led Growth Loop Design

This skill maps and documents a complete PLG loop for a product by identifying viral triggers, activation moments, referral mechanics, and value exchange points that drive self-sustaining user growth.

## Growth Loop Mapping Workflow

### 1. Extract Product Inputs
Ask for or identify:
- What the product does (core value proposition)
- Who the primary user is
- Current acquisition channels
- Any existing sharing, invite, or collaboration features
- The "aha moment" if known

### 2. Identify the Loop Type
Classify the dominant loop mechanic:
- **Viral loop**: Users invite others as part of core usage (e.g., Slack, Figma)
- **Content loop**: User-generated content attracts new users (e.g., Notion templates, Canva designs)
- **Network loop**: Product value increases as more users join (e.g., LinkedIn, Loom)
- **Incentive loop**: Users refer others for explicit rewards (e.g., Dropbox storage, Robinhood stock)
- **Integration loop**: Product embeds into workflows of non-users who then convert (e.g., Calendly links, Typeform forms)

A product may have multiple loops — identify the primary and secondary.

### 3. Map the Four Loop Stages
For each stage, identify the specific in-product moment:

**Stage 1 — Acquisition Trigger**
- What action does an existing user take that exposes the product to a new potential user?
- Is it passive (a Loom video in an email) or active (an explicit invite)?
- What is the surface area where new users first encounter the product?

**Stage 2 — Activation Moment**
- What is the single action a new user must take to experience core value?
- How many steps between first exposure and activation?
- What friction points exist before activation?

**Stage 3 — Value Realization**
- What outcome makes the user want to stay or return?
- Is value delivered to the individual user, or does it require others to participate?
- What is the time-to-value (immediate, days, weeks)?

**Stage 4 — Loop Completion (Re-trigger)**
- What in-product behavior causes the user to naturally expose the product to another new user?
- Is this behavior required for the product to work, or optional?
- What percentage of activated users are likely to complete the loop?

### 4. Identify Loop Leaks
Flag where the loop breaks:
- Long time-to-value (user churns before activation)
- Passive exposure that requires no action from the new user
- Value realization that doesn't create a reason to share
- Missing re-trigger — user gets value but has no reason to spread the product

### 5. Score Loop Strength
Rate each dimension 1–5:
- **Virality coefficient**: How many new users does each activated user expose?
- **Conversion rate**: What fraction of exposed users activate?
- **Cycle time**: How fast does one loop complete?
- **Compounding**: Does each completed loop make the next loop stronger?

### 6. Define Improvement Levers
For each weak stage, propose one concrete mechanic:
- Acquisition trigger improvements: Add share prompts, make output publicly visible, add collaborative features
- Activation improvements: Reduce steps to value, add templates, show social proof
- Value realization improvements: Personalize onboarding, deliver value before sign-up
- Re-trigger improvements: Add collaboration nudges, milestone sharing, in-product invites tied to value
