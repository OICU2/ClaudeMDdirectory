# Product Review Response Strategy

This skill generates a complete review response framework with tone-matched templates for positive, neutral, and negative reviews, giving teams a consistent, brand-aligned playbook for building trust at scale.

## Response Framework Workflow

1. **Gather brand context first.** Ask for: brand name, product category, brand voice descriptors (e.g., warm, professional, playful), any topics that must never be mentioned (legal, competitor names), and the platform(s) where reviews appear (Google, Amazon, Yelp, App Store, etc.).

2. **Define the three response tiers.** Classify reviews before responding:
   - **Positive (4–5 stars):** Grateful, specific acknowledgment. Goal: reinforce loyalty and amplify the signal.
   - **Neutral (3 stars):** Curious, improvement-seeking. Goal: learn and recover the relationship.
   - **Negative (1–2 stars):** Empathetic, action-oriented. Goal: de-escalate publicly, resolve privately.

3. **Build each template with these required components:**
   - **Opening:** Personalize with reviewer name if available; never start with "I."
   - **Acknowledgment line:** Mirror the specific sentiment without parroting the exact words.
   - **Brand value reinforcement:** One sentence that ties the response back to what the brand stands for.
   - **Action or next step:** For neutral/negative, include a direct path to resolution (email, support link, phone).
   - **Closing:** Warm sign-off consistent with brand voice; include team or brand name, never a generic "Management."

4. **Apply platform-specific constraints:**
   - Google/Yelp: Keep under 150 words; no promotional language.
   - Amazon: Cannot solicit positive reviews or offer compensation in the response.
   - App Store: Address the specific feature or bug mentioned; invite re-rating after fix.
   - Social (Facebook, TripAdvisor): Slightly more conversational tone is acceptable.

5. **Flag escalation triggers.** Mark any review containing these signals for human review before responding: legal threats, safety incidents, mentions of illness or injury, media/journalist identification, or repeated profanity.

6. **Create a variation set.** Produce 3 response variants per tier so teams avoid copy-paste repetition, which platforms penalize and customers notice.

## Output Format

Produce a structured document with the following sections:

**Section 1 — Brand Voice Summary** (3–5 bullet adjectives with a one-line do/don't for each)

**Section 2 — Positive Review Templates** (3 variants, each 50–100 words, labeled Variant A/B/C)

**Section 3 — Neutral Review Templates** (3 variants, each 60–110 words, labeled Variant A/B/C, each including a resolution path placeholder like `[support email]`)

**Section 4 — Negative Review Templates** (3 variants, each 70–120 words, labeled Variant A/B/C, each with escalation path and offline resolution CTA)

**Section 5 — Escalation Checklist** (bulleted list of trigger phrases/scenarios that require manager review before posting)

**Section 6 — Usage Guidelines** (5–7 bullets covering: how to personalize templates, what to never copy verbatim, response time SLAs by platform, who owns approval, and how to track response impact)

Format as clean Markdown. Use bold headers for each section. Templates should appear in labeled code blocks for easy copying.
