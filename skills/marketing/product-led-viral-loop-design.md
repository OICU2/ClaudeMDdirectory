# Product-Led Viral Loop Design

This skill analyzes a product's context and outputs a concrete viral loop design with specific mechanics, triggers, incentives, and measurement criteria to drive organic user acquisition.

## Viral Loop Design Process

### 1. Extract Product Context
Identify or ask for:
- Core product category and value proposition
- Current user action that delivers peak value (the "aha moment")
- Existing sharing or social behaviors observed in users
- Business model (freemium, subscription, transactional, marketplace)
- Target user persona and their social environment

### 2. Identify Natural Sharing Moments
Map the three types of organic share triggers:
- **Intrinsic value share**: User shares because the output is inherently shareable (e.g., a report, creation, result)
- **Social proof share**: User shares to signal status, taste, or achievement
- **Collaborative necessity share**: User must invite others to get full value (e.g., splitting a bill, co-editing)

Select the highest-leverage trigger for this product.

### 3. Design the Loop Mechanics
Define each component explicitly:

**Invite Trigger**: The specific in-product moment that prompts the invitation (e.g., after completing first milestone, upon exporting a file, when a streak is reached)

**Invite Mechanic**: The exact method (unique referral link, email invitation, shareable artifact, social post with embedded CTA, copy-invite flow)

**Incentive Structure**: Choose the incentive model:
- Dual-sided reward (both referrer and invitee get value)
- Referrer-only reward (works when social currency is sufficient)
- Feature unlock (gating a premium feature behind invites)
- No incentive if the shareable artifact itself is the hook

**Friction Reduction**: Specify what removes barriers (pre-filled invite messages, one-click share, no account required for invitee to see value)

**Invitee Landing Experience**: Define exactly what the invited user sees first and how it connects back to the referrer's context

### 4. Define the Loop Closure
Specify how the loop closes and compounds:
- What action converts an invitee into an active user
- What triggers the new user to eventually become a referrer themselves
- Time-to-first-invite benchmark to target

### 5. Set K-Factor Inputs
Calculate or estimate:
- **Invites sent per user** (realistic, not aspirational)
- **Conversion rate of invites** (industry benchmark or test hypothesis)
- **K-factor target**: K = invites_sent × conversion_rate (K > 1 = true virality; K > 0.5 = meaningful contribution)

### 6. Identify Failure Modes
Flag the top 3 risks for this specific loop design:
- Incentive abuse vectors (fake accounts, self-referral)
- Invite fatigue patterns
- Drop-off points in the invitee journey

## Output Format

Produce a structured viral loop design document with these exact sections:

**Loop Name**: Short descriptive name for this viral mechanic

**Loop Type**: Intrinsic / Social Proof / Collaborative (with one-line rationale)

**Trigger Moment**: Exact in-product event that initiates the share prompt

**Invite Mechanic**: Step-by-step description of how the invitation works

**Incentive Design**: What each party receives and when they receive it

**Invitee First Experience**: Description of the landing flow with the key hook

**K-Factor Model**:
- Estimated invites per active user: [X]
- Estimated invite conversion rate: [Y%]
- Projected K-factor: [Z]

**Implementation Priority**: 3 ordered development tasks to ship an MVP of this loop

**Failure Mode Mitigations**: Bulleted list of 3 risks and their countermeasures

**Success Metrics**: 4
