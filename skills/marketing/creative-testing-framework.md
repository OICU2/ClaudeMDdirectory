# Creative Testing Framework

This skill builds structured, hypothesis-driven creative testing plans that define variables, audiences, metrics, and success criteria so teams can systematically identify winning ad combinations.

## Creative Testing Workflow

### 1. Define the Testing Goal
- State one primary objective: CTR improvement, conversion rate lift, lower CPA, or brand recall
- Identify the funnel stage: awareness, consideration, or conversion
- Set a minimum detectable effect (e.g., 15% lift in CTR) before building the test

### 2. Isolate Creative Variables
Test one variable at a time per experiment. Common variables:
- **Visual**: static image vs. video vs. carousel vs. GIF
- **Headline**: benefit-led vs. question vs. urgency vs. social proof
- **Copy length**: short (under 50 words) vs. long (100+ words)
- **CTA**: "Shop Now" vs. "Learn More" vs. "Get Started"
- **Color/design**: high-contrast vs. brand-consistent vs. lifestyle vs. product-only
- **Format**: square (1:1) vs. vertical (9:16) vs. landscape (16:9)

### 3. Build the Test Matrix
Create a structured table with:
| Variable | Control (A) | Variant (B) | Variant (C) |
|---|---|---|---|
| Visual type | Static image | Video | Carousel |
| Headline | Benefit-led | Question | Urgency |
| CTA | Shop Now | Learn More | — |

- Limit to 2–3 variants per variable to maintain statistical clarity
- Document the hypothesis for each variant: "We believe [variant] will outperform [control] because [reason]"

### 4. Define Audience & Budget Splits
- Split budget equally across variants (e.g., 50/50 or 33/33/33)
- Use identical audience targeting for all variants — no audience mixing
- Minimum recommended spend per variant: enough to reach 1,000+ impressions or 100+ clicks
- Run duration: minimum 7 days to account for day-of-week variance

### 5. Set Success Metrics
Define primary and secondary KPIs before launch:
- **Primary**: CTR, conversion rate, or CPA (pick one)
- **Secondary**: video completion rate, engagement rate, or ROAS
- Set statistical significance threshold: 95% confidence minimum
- Define what "winning" looks like before the test ends

### 6. Analyze and Document Results
- Compare variants against control on primary KPI only
- Note performance by placement (feed vs. stories vs. reels) separately
- Record losing variants — document why they underperformed
- Apply winner immediately; feed learnings into the next test hypothesis

## Output Format

Produce a complete creative testing plan with the following sections:

1. **Test Overview** — Goal, funnel stage, platform, timeline (3–5 sentences)
2. **Hypothesis Table** — One row per variant with: Variable | Control | Variant | Hypothesis
3. **Test Matrix** — Full grid of all creative combinations being tested
4. **Audience & Budget Split** — Targeting parameters, budget per variant, flight dates
5. **KPI Dashboard** — Primary metric, secondary metrics, significance threshold, winner criteria
6. **Results Log Template** — Pre-formatted table with columns: Variant | Impressions | Clicks | CTR | Conversions | CVR | CPA | Winner (Y/N) | Notes

Format as structured markdown with tables. Length should match complexity — a simple 2-variable test needs 1 page; a multivariate test needs 2–3 pages. Always end with a "Next Test Recommendation" line based on the variables not yet tested.
