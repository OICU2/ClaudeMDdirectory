# Pricing Page Optimization

This skill audits and rewrites a pricing page's layout, tier logic, copy, and calls-to-action to reduce cognitive load and drive users toward the intended plan selection.

## Audit & Restructure Workflow

1. **Identify the anchor plan**: Determine which plan generates the most revenue or is most strategically important. All layout and copy decisions should frame other tiers relative to this one.

2. **Audit tier logic**: Verify tiers follow a clear value ladder — each step up must justify its price with 2-3 concrete, meaningful feature differences. Flag tiers that overlap in value or confuse the upgrade path.

3. **Apply visual hierarchy**: Recommend the anchor plan be visually elevated (highlighted border, badge like "Most Popular" or "Best Value", slightly larger card). Flanking plans should feel like natural comparisons, not distractions.

4. **Rewrite tier names**: Replace generic labels (Basic/Pro/Enterprise) with outcome-based or identity-based names that signal who the plan is for (e.g., Starter / Growth / Scale, or Solo / Team / Agency).

5. **Tighten feature lists**: Each tier should list 5-8 features maximum. Lead with the highest-value differentiators. Remove redundant items that appear identically across all tiers — they add noise, not value.

6. **Rewrite CTAs**: Each plan needs a distinct, action-oriented CTA. Avoid repeating "Get Started" across all tiers. Use copy that matches intent: "Start Free," "Start Growing," "Talk to Sales."

7. **Add friction reducers**: Below each CTA, include one trust signal — no credit card required, free trial length, cancel anytime, or money-back guarantee. These must be tier-appropriate.

8. **Address the enterprise edge case**: If an enterprise or custom tier exists, replace the price with a "Contact Us" CTA and list 3-4 specific enterprise-only benefits (SSO, SLA, dedicated support) rather than vague "everything in Pro plus more" language.

9. **Write a pricing header**: Provide a 1-sentence headline and 1-sentence subheadline for the top of the pricing section. The headline should state the value proposition; the subheadline should neutralize price anxiety (e.g., "No contracts. Change plans anytime.").

10. **Identify FAQ gaps**: List 3-5 questions users likely ask before converting (e.g., "Can I switch plans later?" "What happens when I hit my limit?") and draft concise answers for a pricing FAQ section.

## Output Format

Produce a structured optimization report with the following sections:

- **Audit Summary** (3-5 bullet points identifying the current page's conversion blockers)
- **Recommended Tier Structure** (table with: Tier Name | Target User | Price | 5-8 Key Features | CTA Copy | Trust Signal)
- **Rewritten Copy** (pricing section headline + subheadline, all tier names, all CTA copy)
- **Visual Hierarchy Recommendations** (2-4 specific layout or design directives, written as clear instructions for a developer or designer)
- **Pricing FAQ** (3-5 Q&A pairs, each answer under 2 sentences)
- **Priority Changes** (ranked list of top 3 changes most likely to lift conversion, with one-line rationale each)

Total output should be comprehensive but scannable — use tables and bullet points throughout, not prose paragraphs.
