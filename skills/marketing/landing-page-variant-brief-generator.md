# Landing Page Variant Brief Generator

This skill produces complete, structured briefs for multiple landing page variants so teams can build, brief designers, or brief copywriters without starting from scratch.

## Workflow

1. **Gather inputs** — Ask for (or extract from context): product/service name, primary offer, known audience segments or traffic sources, core value proposition, and any existing conversion goals or metrics.

2. **Define variants** — Identify 2–5 distinct variants based on:
   - Audience segment (e.g., SMB vs. enterprise, beginner vs. expert)
   - Traffic source (e.g., paid social, Google Ads, email, organic)
   - Pain point angle (e.g., cost-focused, time-focused, risk-focused)
   - Funnel stage (e.g., awareness, consideration, retargeting)

3. **Build each variant brief** using the structure below. Repeat for every variant.

4. **Flag conflicts or gaps** — Note any variants that overlap too heavily in messaging or where input data is insufficient to differentiate meaningfully.

5. **Recommend priority order** — Based on likely traffic volume or conversion impact, suggest which variant to build first.

## Output Format

Produce one clearly labeled section per variant. Each variant brief must include:

---

### Variant [N]: [Descriptive Name]

**Target Audience:** [One sentence — who this page is for, including psychographic or demographic detail]

**Traffic Source / Entry Point:** [Where this visitor is coming from]

**Primary Pain Point Addressed:** [The specific problem this variant leads with]

**Hero Headline:** [Draft headline, 8–12 words]

**Supporting Subheadline:** [Draft subheadline, 15–25 words]

**Value Proposition (3 bullets):**
- [Benefit 1 — outcome-focused]
- [Benefit 2 — outcome-focused]
- [Benefit 3 — outcome-focused]

**Primary CTA Text:** [Button or form label copy]

**Social Proof Angle:** [What type of proof works best here — testimonials, logos, stats, case studies — and why]

**Tone & Voice:** [2–3 adjectives + one sentence describing register]

**Key Messaging Differences from Other Variants:** [What makes this brief distinct — 1–2 sentences]

**Conversion Goal:** [Specific action: form fill, demo request, purchase, download, etc.]

---

After all variants, include:

**Variant Matrix Summary** — A plain-text table mapping each variant against: Audience, Traffic Source, Pain Point, and CTA.

**Recommended Build Priority:** Numbered list with one-line rationale per variant.
