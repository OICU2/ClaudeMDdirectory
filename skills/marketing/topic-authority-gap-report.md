# Topic Authority Gap Report

This skill audits a site's content coverage against competitors and top-ranking pages to surface specific topic areas where authority content is absent, thin, or underdeveloped, giving content teams a prioritized list of gaps to close.

## Workflow

### 1. Establish Scope
- Confirm the target domain and 2–5 competitor domains to benchmark against
- Define the primary topic cluster or keyword universe (e.g., "B2B payroll software," "sustainable packaging")
- Clarify the goal: ranking improvement, topical authority, audience coverage, or all three

### 2. Inventory the Site's Existing Content
- Catalog published URLs by topic, format, and funnel stage (awareness / consideration / decision)
- Flag thin content: pages under ~500 words, duplicate-intent pages, or pages with no backlinks and no ranking position
- Group URLs into topic buckets using the target keyword universe as the taxonomy

### 3. Map Competitor Content Coverage
- For each competitor domain, identify content covering the same topic universe
- Note topics where competitors have 3+ pieces of content and the target site has 0–1
- Identify competitor cornerstone or pillar pages that have no equivalent on the target site
- Record approximate depth signals: word count ranges, presence of original data, multimedia, structured data

### 4. Analyze Top-Ranking Pages for Target Topics
- For each priority topic, identify the top 3–5 SERP results
- Extract the subtopics, angles, and content formats those pages use
- Compare against what the target site has published on the same topic
- Flag intent mismatches: target site ranks a blog post where a tool or comparison page dominates

### 5. Score and Prioritize Gaps
Rank each identified gap using these four dimensions:
- **Search volume / business value** (high / medium / low)
- **Competitive difficulty** (how entrenched are top-ranking pages)
- **Gap severity** (missing entirely vs. thin vs. wrong format)
- **Topical adjacency** (how close to existing content clusters — easier to build authority on adjacent topics)

Assign each gap a priority tier: **P1 (act now)**, **P2 (next quarter)**, **P3 (backlog)**.

### 6. Identify Root Causes
For each P1 and P2 gap, note why the gap likely exists:
- Topic never addressed
- Content exists but targets wrong intent
- Content exists but is outdated or too shallow
- Content exists but is siloed with no internal links

## Output Format

Produce a structured report with the following sections:

---

**Executive Summary** (3–5 bullets)
- Total topic gaps identified
- Number of P1 gaps requiring immediate action
- Key pattern or insight (e.g., "Site has strong top-of-funnel coverage but zero comparison or alternative pages")

---

**Gap Inventory Table**

| Topic / Subtopic | Target Site Coverage | Competitor Coverage | Top SERP Format | Gap Severity | Priority |

Include 10–30 rows depending on scope. Sort by Priority then Gap Severity.

- **Recommended content type:** pillar page / comparison / FAQ cluster / original research / etc.
- **Root cause:** missing / wrong intent / outdated / isolated
