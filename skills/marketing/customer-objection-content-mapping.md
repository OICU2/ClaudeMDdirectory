# Customer Objection-to-Content Mapping

This skill takes a list of real sales objections and maps each one to specific content types, existing assets, and recommended funnel placement so marketing can directly support sales conversations.

## Mapping Workflow

1. **Collect objections**: Ask the user to provide their top objections. If none are supplied, prompt: "List your 5–10 most common sales objections — exact language from prospects is ideal."

2. **Categorize by objection type**: Sort each objection into one of these buckets:
   - Price/ROI ("too expensive," "can't justify the cost")
   - Trust/Credibility ("never heard of you," "how do I know this works")
   - Timing ("not the right time," "maybe next quarter")
   - Authority ("need to run this by my boss")
   - Competitor ("we're already using X")
   - Fit ("not sure this applies to our industry/size")
   - Complexity/Risk ("seems hard to implement," "what if it doesn't work")

3. **Assign funnel stage**: For each objection, identify where in the funnel it typically surfaces:
   - TOFU (awareness): prospect doesn't know enough yet
   - MOFU (consideration): prospect is evaluating options
   - BOFU (decision): prospect is close but hesitating

4. **Map to content types**: For each objection, recommend 2–3 content formats that directly address it:
   - Price/ROI → ROI calculators, case studies with metrics, TCO comparison guides
   - Trust/Credibility → customer testimonials, analyst reviews, awards/certifications, founder story
   - Timing → urgency-driven case studies, cost-of-inaction one-pagers, limited offer landing pages
   - Authority → executive summary docs, stakeholder presentation templates, approval guides
   - Competitor → battlecards, neutral comparison guides, "why switch" testimonials
   - Fit → industry-specific case studies, use case libraries, vertical landing pages
   - Complexity/Risk → implementation guides, onboarding timelines, risk reversal offers, pilot program briefs

5. **Audit against existing assets**: Ask if the user has existing content. If yes, map current assets to objections and flag gaps. If no, mark all as "content needed."

6. **Identify top 3 gaps**: Highlight the objections with the weakest content coverage — these become priority content briefs.

## Output Format

Produce a structured objection-content matrix with the following format:

---

**Objection:** [Exact or paraphrased objection]
**Type:** [Objection category]
**Funnel Stage:** [TOFU / MOFU / BOFU]
**Recommended Content Types:** [2–3 specific formats]
**Existing Assets:** [Asset name/link or "None identified"]
**Gap Status:** [Covered / Partially covered / Gap — content needed]

---

Repeat for each objection. After the full matrix, include:

**Top 3 Content Gaps to Prioritize:**
1. [Objection] — [Recommended content type] — [Why it matters]
2. [Objection] — [Recommended content type] — [Why it matters]
3. [Objection] — [Recommended content type] — [Why it matters]

**Suggested Next Step:** One sentence recommending whether to brief a writer, audit the CMS, or schedule a sales-marketing alignment meeting.

Total output length scales with number of objections — expect 1 structured block per objection plus a 3-item priority list.
